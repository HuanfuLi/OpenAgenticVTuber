## Architectural pivot vs original plan

The plans were written assuming a **shadcn + Radix + Tailwind v4** UI stack. The prototype proves that approach is unnecessary — the chrome is small enough that hand-rolled CSS + inline SVGs ship in **~840 lines of CSS + 17 SVGs + 11 component files** vs. the ~30 npm packages + boilerplate config a shadcn install adds. **The prototype's approach is what ships.**

## Conversion rules (apply to every prototype file)

1. **React 18 → React 19.** Keep `useState`/`useEffect`/`useMemo`/`useCallback`/`useRef`/`createContext`/`useContext` — APIs are unchanged. Drop `React.` prefix; rely on automatic JSX runtime (`"jsx": "react-jsx"` in tsconfig).
2. **JSX → TSX.** Add prop types for every component (`interface XProps { ... }` or inline `({ size = 16 }: { size?: number })`).
3. **`window.X = Y` registry → ESM `export`.** Every `window.COPY = {...}` / `window.ICONS = {...}` / `window.TopBar = TopBar` line in the prototype becomes an `export` statement; consumers import explicitly.
4. **Babel/standalone → Vite.** No `<script type="text/babel">`. All files become real ES modules under Vite. Imports resolve through `tsconfig.json` `paths` (`@/*` → `apps/renderer/src/*`).
5. **`window.MOCK.*` → real backend.**
   - `mockEcho(text)` → `wsClient.send({type:'text-input', text})` and await `display-text` reply.
   - `mockSafeStorage.get(k)` / `mockSafeStorage.set(k, v)` → `window.api.getStoredValue(k)` / `window.api.setStoredValue(k, v)` (async; safeStorage IPC).
   - `mockTestConnection(...)` async generator → SSE consumer reading from `/admin/llm-test` over the sidecar HTTP server.
   - `mockStatus.subscribe(cb)` → real status events from the sidecar over WS (out of Phase 1 scope; keep as DEV mock for now, replace in Phase 4).
   - `startSidecarLogs(cb)` → real stdout-line events forwarded from main via `window.api.onSidecarLog(cb)`.
6. **DevPanel preservation.** `src/dev/DevPanel.jsx` and `src/lib/mock.js` get ported as-is into `apps/renderer/src/dev/` and `apps/renderer/src/dev/__mocks__/`. Mount the DevPanel in `App.tsx` with `{import.meta.env.DEV && <DevPanel />}`. Production builds tree-shake the dev modules.
7. **Drop the macOS title-bar dots.** Lines 23–26 of `src/App.jsx` (and similar in `GatedAppShell`) render fake macOS window-control dots. Electron supplies real OS chrome — drop these spans.
8. **`completeSetup` is now async.** Lines 127–131 of `src/lib/store.jsx` need `await window.api.setStoredValue(...)` calls; update the callsite in `LLMSetup.jsx` (line 105) to handle the promise.
