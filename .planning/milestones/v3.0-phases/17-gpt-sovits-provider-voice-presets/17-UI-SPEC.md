# Phase 17 — UI Design Contract

> Visual and interaction contract for GPT-SoVITS provider selection, setup, voice presets, reference-audio import, test synthesis gating, and visible failure/fallback states. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Phase UI Scope

Phase 17 extends the existing `Settings` long-scroll surface, specifically `#sec-tts`, and may add only compact chat/playback failure affordances needed to mark GPT-SoVITS audio failures. It is **not** a new standalone brand or settings shell.

**In scope**
- Select active TTS provider: Piper or GPT-SoVITS.
- Configure GPT-SoVITS base URL plus optional app-launched mode using user-provided command, working directory, health URL, and start/stop/restart controls.
- Run health check and user-triggered test synthesis before GPT-SoVITS can become active.
- Create, rename, select, and delete named voice presets in a global reusable library.
- Import GPT-SoVITS reference audio into app-managed storage, capture required transcript text and language, show validation failures, and block unsafe deletion.
- Show concise GPT-SoVITS failure/fallback state in Settings and chat; route technical detail to the existing log panel.

**Out of scope**
- STT provider UI, microphone capture, PTT/VAD, code-switch evaluation, AEC/no-headphones controls, GPT-SoVITS installer/training/voice-cloning UI, and avatar catalog mutation for active voice selection.

Sources: `17-CONTEXT.md` decisions D-01 through D-16, `.planning/REQUIREMENTS.md` TTS-01..06 and PRESET-01..04, user constraints.

## Spacing Scale

Declared values for **new Phase 17 UI** (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline status dot/text gaps, compact helper spacing |
| sm | 8px | Row padding, button gaps, form field internal gaps, banner padding |
| md | 16px | Settings scroll padding, dialog padding, card padding, default section internals |
| lg | 24px | Anchor-to-section separation, major form group separation |
| xl | 32px | Section bottom margin |
| 2xl | 48px | Not used in this phase unless a future empty state requires vertical centering |
| 3xl | 64px | Not used in this phase |

Exceptions: existing grandfathered CSS uses 6px in anchor/field helper gaps; do not introduce additional 6px values in Phase 17. Touch/click targets remain at least 36px for standard controls; icon-only controls must be at least 32px and keyboard focusable, matching existing `.icon-btn`.

## Color

Use existing CSS variables so light/dark/accent themes continue to work.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--background)` | Settings page background, chat background, default section surface |
| Secondary (30%) | `var(--card)` / `var(--popover)` | Form controls, grouped preset cards, dialog surface, test log, logs drawer |
| Accent (10%) | `var(--primary)` | Active provider/preset radio state, primary CTA, focus ring, active anchor pill, enabled switch only |
| Destructive | `var(--destructive)` | Delete preset, blocked/deletion confirmations, GPT-SoVITS failed sentence/audio state, validation errors that prevent activation |

Accent reserved for: selected provider, selected preset, successful activation CTA, focus-visible outlines, active switches, active anchor pill, and progress/success emphasis in test synthesis. Accent must not be used for generic secondary buttons or all links.

Semantic status colors:
- `var(--success)` green dot: health + test synthesis passed.
- `var(--warning)` amber dot/banner: reachable but not activated, fallback available next turn, external service warning, candidate draft not tested.
- `var(--destructive)` red dot/banner: health/test failed, audio failed for a sentence, blocked deletion/invalid reference audio.

## Copywriting Contract

All user-visible strings must live in `apps/renderer/src/lib/copy.ts`, following the existing `COPY.SETTINGS` pattern.

| Element | Copy |
|---------|------|
| Primary CTA | `Test synthesis` |
| Secondary activation CTA | `Activate voice preset` — enabled only after health check and test synthesis pass |
| Empty state heading | `No voice presets yet.` |
| Empty state body | `Create a preset to test GPT-SoVITS with reference audio before using it in chat.` |
| Error state | `GPT-SoVITS test failed. Your current voice settings were not changed. Open logs for details, then check the server and reference audio.` |
| Destructive confirmation | Delete preset: `Delete voice preset? Choose another active preset before deleting this one.` |
| Destructive confirmation | Delete reference audio: `Reference audio is used by {N} preset(s). Reassign those presets before deleting it.` |
| Destructive confirmation | Stop app-launched GPT-SoVITS: `Stop app-launched GPT-SoVITS? This only stops the process AgenticLLMVTuber started.` |

Required concise status copy:
- Provider not ready: `Run health check and test synthesis before activating GPT-SoVITS.`
- Health passed, test pending: `Server is reachable. Test synthesis is still required.`
- Activation success: `GPT-SoVITS is active for the next turn.`
- Candidate failure: `Test failed. Keeping the previous provider and preset.`
- Mid-turn audio failure: `Audio failed for this sentence. Text is still shown.`
- Next-turn fallback notice: `GPT-SoVITS failed. Select Piper for the next turn if you want fallback audio.`
- Reference path failure: `GPT-SoVITS could not read the copied reference audio. Open logs for details.`

Do not show raw stack traces, full provider stderr, original absolute user file paths, or full reference transcripts in the main UI. Use `Open logs` / existing log drawer for diagnostic details.

## Interaction Contract

### Settings layout

- Keep the current single long-scroll Settings page and anchor pills.
- Keep the existing anchor label `TTS`; do not add a separate `Voice presets` top-level page in Phase 17.
- Primary visual anchor is the GPT-SoVITS activation/test status group inside the TTS section: provider state, health result, `Test synthesis`, and `Activate voice preset` should draw the eye before secondary preset/reference-audio management controls.
- Replace the current compact Piper-only `TTSSection` contents with progressive groups in this order:
  1. **Provider** — radio rows for `Piper local TTS` and `GPT-SoVITS`, capability labels, and provider health dot.
  2. **GPT-SoVITS setup** — base URL, connection mode, optional app-launched command/cwd/health URL, health/test controls, concise status.
  3. **Voice presets** — selected preset radio/list, create/rename/delete, active association note.
  4. **Reference audio** — import/manage reference audio, transcript text, language, validation summary.
  5. **Output invariant summary** — existing output device and lipsync rows to reassure that ordered playback/RMS/VTS path remains unchanged.

### Activation gate

- Selecting GPT-SoVITS or changing a GPT-SoVITS preset creates a candidate state.
- Candidate GPT-SoVITS cannot become active until both health check and test synthesis pass.
- Failed health or failed test synthesis leaves the previous active provider/preset intact and shows a red/amber concise notice.
- `Activate voice preset` is disabled until the gate passes and must include a helper line explaining the missing gate.

### Provider controls

- External-server mode asks for one base URL only.
- App-launched mode asks for user command and working directory; no templates or installer affordance.
- `Stop` and `Restart` controls are visible only for app-launched processes started by this app. External servers show `External server — stop it outside AgenticLLMVTuber.`
- Non-localhost base URLs may be allowed but must show amber copy: `Advanced: this sends text/reference paths to a non-local GPT-SoVITS server.`

### Preset CRUD

- Presets are a global reusable library, visually represented as radio/list rows with name, provider, language, and validation state.
- Creating a preset opens an inline form or existing dialog style; primary action is `Save preset`, followed by `Test synthesis`.
- Renaming uses an inline edit pattern consistent with history/session edit tests where possible.
- Deleting an active preset is blocked until the user selects/reassigns another active preset; never silently switch to Piper.

### Reference-audio import

- `Import reference audio` opens the native file picker and then shows copied managed asset metadata, not the original absolute path.
- Required fields: reference transcript text and language.
- Validation summary uses short labels: `Format`, `Duration`, `Metadata`, `Server access`.
- Validation failures are visible before activation and block `Activate voice preset`.
- Deleting reference audio used by presets is blocked; do not cascade-delete presets.

### Failure/fallback state

- GPT-SoVITS chat-turn failure must mark the affected sentence/audio as failed while leaving sentence text visible.
- The app must not auto-switch to Piper mid-turn.
- Piper fallback is a next-turn explicit selection/action only, after visible notice.
- Main UI shows concise status plus `Open logs`; diagnostics detail remains in the existing logs drawer/log folder.

## Accessibility Contract

- Provider and preset choices use `role="radiogroup"` / `role="radio"` or native radio inputs with `aria-checked`, matching existing `RadioRow` behavior.
- App-launched controls expose clear button names: `Start GPT-SoVITS`, `Stop app-launched GPT-SoVITS`, `Restart app-launched GPT-SoVITS`.
- Candidate/active/failed state must be available as text, not color alone.
- Disabled activation buttons must include nearby helper copy explaining exactly which gate is missing.
- Destructive dialogs use `role="alertdialog"`, existing `.dialog` visual treatment, and explicit cancel/confirm buttons.
- Focus ring remains existing `2px solid var(--primary)`.
