## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-17-08-01 | Tampering | Preset validation evidence | mitigate | Compute deterministic fingerprint over synthesis-affecting provider/preset fields and require exact match before treating stored evidence as valid. |
| T-17-08-06 | Tampering | Renderer/main validation computation drift | mitigate | Put fingerprint/state field selection in `packages/contracts/ts/gpt-sovits-validation.ts` and require renderer plus Electron main tests to import the same source module. |
| T-17-08-02 | Repudiation | Test synthesis proof | mitigate | Store timestamped per-preset validation metadata only after successful test synthesis; failed tests do not create proof. |
| T-17-08-03 | Information Disclosure | Validation summary/UI | mitigate | Keep validation summaries redacted and copy-only; do not include raw reference paths, transcripts beyond existing preset fields, or provider diagnostics in validation labels. |
| T-17-08-04 | Denial of Service | Preset switch sidecar restart | accept | Restart is an explicit local settings action needed to apply runtime config; existing sidecar restart error path surfaces failure. |
| T-17-08-05 | Elevation of Privilege | App-managed launch fields in fingerprint | mitigate | Fingerprint launch.command/cwd only for invalidation; do not execute, mutate, or template commands in this plan beyond existing app-owned lifecycle controls. |
</threat_model>
