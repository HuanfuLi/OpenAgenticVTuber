<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| renderer settings → Electron/main storage → sidecar env/admin API | User-entered weight paths become persisted local configuration and are sent to sidecar health/test flows. |
| sidecar → GPT-SoVITS localhost HTTP server | Sidecar sends configured weight paths to an external local service over HTTP. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-17-11-01 | Tampering | `GptSoVitsPresetConfig` weight paths | mitigate | Treat paths as explicit user configuration and include them in validation fingerprint so activation is invalidated when paths change. |
| T-17-11-01b | Tampering | `GptSoVitsPresetConfig.text_lang` | mitigate | Treat synthesized text language as synthesis-affecting configuration and include it in validation fingerprint so activation is invalidated when it changes. |
| T-17-11-02 | Information Disclosure | Settings, UAT, provider errors | mitigate | UI may show user-entered paths as configuration, but provider errors should redact to endpoint/status/message and avoid dumping external server stack traces. |
| T-17-11-03 | Denial of Service | `/set_gpt_weights`, `/set_sovits_weights`, `/tts` calls | mitigate | Reuse existing `request_timeout_ms`; timeout maps to typed GPT-SoVITS failure and does not trigger Piper fallback. |
| T-17-11-04 | Spoofing | GPT-SoVITS server at base URL | accept | Phase 17 is localhost/external-server explicit; health and set-weight calls prove reachable API behavior, not server identity. |
| T-17-11-05 | Repudiation | UAT completion state | mitigate | Keep UAT partial/testing and record Plan 17-11 automated evidence plus manual retest criteria before complete status can be reclaimed. |
</threat_model>
