# 20-06 Plan: Truthful STT Model Cache And VAD Copy

<objective>
Remove the fake local STT model download behavior and make the Voice settings UI truthful: local model status must reflect real model availability, and VAD controls must be clearly model-free and separate from STT model cache controls.
</objective>

<execution_context>
The live user report identified the current model download behavior as mock-like. The audit confirmed the sidecar endpoint creates a placeholder directory and immediately reports `downloaded`. This gap closure must either implement a real provider-specific download or, more conservatively, remove/disable/rename the action so the app no longer claims a model has been downloaded when it has not.

Prefer the conservative fix unless real provider downloads can be implemented safely with progress, cancellation/error states, and provider-specific validation inside this plan's scope.
</execution_context>

<tasks>

## 1. Replace Placeholder Download Semantics

Update `STTModelCache` and the `/admin/audio/stt/models/download` endpoint so placeholder creation no longer reports `downloaded`.

Conservative acceptable behavior:

- endpoint returns `ok: false` or a non-downloaded status with summary like `Automatic download is not implemented yet. Place the model in the app-managed cache or configure a local model path.`;
- no placeholder marker is created;
- existing placeholder marker directories from earlier runs are not treated as downloaded;
- remove still safely removes app-managed cache entries.

If implementing real download instead:

- use provider-specific official download APIs or cache mechanisms;
- surface real failures and partial state;
- do not block app startup;
- verify actual model files exist before returning downloaded.

## 2. Strengthen Local Model Presence Detection

Change model-cache catalog/local status logic so `downloaded` means actual usable model contents are present.

Rules:

- a directory containing only `.agenticllmvtuber-model-cache.json` is not downloaded;
- empty directories are not downloaded;
- for `local_model_path_override`, readiness may use the explicit path if it exists and looks non-empty;
- local provider runtime transcription remains blocked when model status is not downloaded.

Add sidecar tests for placeholder-only directory, empty directory, and real non-empty cache marker/file behavior.

## 3. Make Settings Model Cache UI Truthful

Update Voice settings local model cache UI:

- label it as STT local model cache, not generic voice/VAD model;
- if automatic download is unavailable, disable or relabel the button accordingly;
- display the sidecar summary from the failed/unavailable operation;
- do not show instant success for placeholder behavior;
- keep Remove available only for real app-managed cache entries.

Add Settings tests that clicking the action no longer produces a fake downloaded state and that the UI text distinguishes STT model cache from VAD controls.

## 4. Clarify VAD Is Model-Free

Update Settings and Chat copy as needed:

- VAD is browser volume/silence detection;
- VAD has sensitivity and silence timeout controls;
- VAD does not require or download a model;
- no-headphones/AEC remains deferred to Phase 22.

Keep copy concise and operational.

## 5. Update UAT And Verification Notes

Update Phase 20 UAT artifacts to add checks for:

- sidecar startup readiness recovery from `Sidecar is not ready`;
- successful STT test + save enabling Chat PTT;
- model-cache UI no longer claiming fake instant download;
- VAD copy clearly separate from STT model cache.

</tasks>
