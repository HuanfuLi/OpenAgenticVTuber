# 20-10 Plan: STT Model Removal UX And Redownload

## Goal

Removing a selected local STT model should invalidate Chat readiness without looking like the user clicked the primary Save settings button, and the Voice input page must keep the STT local model cache card visible so the user can immediately download the model again.

## Root Cause

`removeSelectedModel()` calls `persistVoiceInput(invalidatedConfig)`, which is the same path used by the Save settings button. That shows generic save-status copy and runs full voice-input save side effects for a cache operation. The cache card is rendered only when `selectedModel` exists, so a transiently missing/stale refreshed catalog entry after removal hides the card and its Download button until the Voice input panel remounts and reloads the catalog.

## Implementation Tasks

1. Split cache-operation persistence from Save settings UX.
   - Add a focused helper that persists the invalidated STT readiness/config without using generic Save settings success copy.
   - After successful removal, show the remove operation summary or a model-cache-specific message, not `Voice input enabled. Sidecar restarted.` / `Voice input settings saved.`
   - Preserve the existing Chat readiness refresh/config-changed event.

2. Keep the local model cache card stable after removal.
   - Ensure the active local provider always has a visible model-cache entry even if the refreshed catalog is briefly missing the selected provider.
   - Represent the removed model as `not_downloaded`, `removable: false`, and keep Download enabled.
   - Avoid requiring category navigation/remount to re-fetch the card.

3. Tighten blocking behavior.
   - Removing a downloaded active local model should invalidate readiness with `missing_model`.
   - Chat should receive the config-changed/readiness refresh so PTT blocks before recording starts.
   - Do not silently change STT provider or active model.

4. Add regression coverage.
   - Extend `Settings.test.tsx` so Remove model does not surface generic Save settings copy.
   - Assert the STT local model cache card remains visible after removal and the Download model button remains available.
   - Assert missing-model readiness invalidation is still persisted and `notifyVoiceInputConfigChanged` behavior is preserved.

## Verification

- `npm --workspace apps/renderer run test -- --run Settings`
- `npm --workspace apps/renderer run typecheck`

## Out Of Scope

- Do not change download provider behavior or add automatic background download.
- Do not make model removal a global configuration save action.
- Do not change cloud STT settings.
