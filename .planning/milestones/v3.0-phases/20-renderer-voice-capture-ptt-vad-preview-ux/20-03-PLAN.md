# 20-03 Plan: Chat PTT Preview, Final Submit, And Safe Queue

<objective>
Integrate the voice capture controller into Chat: visible PTT-first mic control, transient transcript preview, final transcript submission through the existing typed-chat path, and safe queueing while a turn is active.
</objective>

<execution_context>
This plan depends on 20-02 capture/store work. Preserve the existing typed chat behavior in `Chat.tsx`, `useStreamingMessages.ts`, and `ws/store.ts`. Voice final submission should be a thin caller of the existing send path, not a separate conversation route.
</execution_context>

<tasks>

## 1. Extract Shared Final Text Submit Helper

Refactor `Chat.tsx` enough to let typed input and final voice transcript use the same submission helper.

The helper must:

- trim and reject empty text;
- append the user bubble through existing `appendUserMessage`;
- send `type: text-input` with unchanged text, active session id, and active session history;
- keep existing input-disabled and banner behavior.

Do not change the sidecar `TextInputMessage` contract unless 20-01 already required a compatible extension.

## 2. Add Chat Voice Input Control

Create `apps/renderer/src/screens/Chat/VoiceInputControl.tsx`.

UI behavior:

- visible beside the text input row as an icon control;
- disabled with setup link/path when STT readiness is blocked;
- requests permission on first use if needed;
- hold mic button to record PTT;
- uses configured PTT shortcut from Settings;
- shows idle, listening, recording, previewing, finalizing, queued, and error states.

Keep text compact and avoid in-app instructional prose beyond state labels and setup/error copy.

## 3. Render Transient Transcript Preview

Show preview transcript near the input row, not as a chat bubble.

Rules:

- preview chunks/results update only the voice input store;
- preview text never calls `appendUserMessage`;
- preview text never calls `commitConversationTurn`;
- finalizing state is distinct from a submitted user bubble;
- cancel clears preview/final candidate and stops mic tracks.

## 4. Submit Final Transcript Through Existing Chat Path

On final STT result:

- send final transcript unchanged through the shared submit helper;
- do not translate or rewrite mixed-language text;
- clear preview state after successful local submission;
- surface transcription errors as voice input errors, not assistant/LLM stream errors.

## 5. Queue During Active Turn

If `turnInFlight`, `inputDisabled`, or `isSpeaking` indicates an active turn:

- do not send immediately;
- store one visible pending utterance with cancel/send-later state;
- auto-send after `conversation-chain-end` only if still connected and not canceled;
- if another utterance is captured while one is queued, show a clear blocked/replace state rather than corrupting the queue.

Do not cancel active TTS or implement barge-in.

## 6. Add Chat Voice Tests

Add `apps/renderer/tests/ChatVoiceInput.test.tsx` and extend existing Chat tests.

Cover:

- mic control is visible but disabled when STT is not ready;
- setup action routes to Settings/Voice Input;
- hold-to-talk calls voice capture start/stop;
- preview transcript renders outside chat bubbles;
- preview transcript does not call conversation history APIs;
- final transcript sends the same `text-input` envelope as typed chat;
- final transcript includes active session history;
- final text is unchanged for mixed Chinese/English input;
- active turn queues final transcript until chain-end;
- queued transcript can be canceled and does not affect active TTS/playback state.

</tasks>
