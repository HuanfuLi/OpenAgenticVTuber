# 20-09 Plan: Active-Turn Voice Queue Ordering

## Goal

Queued voice input captured during an active Teto turn must dispatch only after the current turn is durably represented in chat history and streaming cleanup has removed its transient rows. The visible conversation must become first question, first answer, queued second question, second answer with no duplicated first turn and no dropped queued turn.

## Root Cause

`conversation-chain-end` starts async conversation persistence, then immediately clears `inputDisabled`. That allows `Chat.tsx` to promote a queued voice final candidate before the first turn has been committed and before `markCompletedTurnConsumed()` removes transient streaming rows. The queued dispatch overwrites the module-level `pendingTurn`; when the first commit resolves, cleanup consumes the wrong pending turn and leaves the duplicated first Q/A while dropping the queued second question.

## Implementation Tasks

1. Make streaming turn completion identity-aware.
   - Extend the completed-turn candidate with the originating `userMessageId`.
   - Change `markCompletedTurnConsumed` to accept that id and remove only the matching completed turn.
   - If the current pending turn no longer matches, leave the newer pending turn intact.

2. Add a turn-settling guard for queued voice dispatch.
   - Expose a small streaming state flag or selector that remains true between `conversation-chain-end` and successful persistence/cleanup.
   - Keep `VoiceInputControl` queue visibility unchanged, but prevent queued dispatch while the previous turn is settling.
   - Clear the settling flag on successful cleanup and on error paths that already re-enable input.

3. Preserve correct history for queued send.
   - Ensure the queued text is sent after `activeSession.messages` includes the just-committed first turn.
   - Avoid using stale pre-commit history for queued voice dispatch.

4. Add regression coverage.
   - In `ChatVoiceInput.test.tsx`, simulate first VAD/user turn, assistant response, queued PTT final while `isSpeaking/inputDisabled` are active, `conversation-chain-end`, and a deferred `commitConversationTurnFromDispatcher` promise.
   - Assert the queued send does not fire until the commit resolves and streaming cleanup completes.
   - Assert the visible chat sequence has no duplicated first question/answer and contains the queued second question.
   - Assert the outbound queued `text-input` history includes the first Q/A turn.
