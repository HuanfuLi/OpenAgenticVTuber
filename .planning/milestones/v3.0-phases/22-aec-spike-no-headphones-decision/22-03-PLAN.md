# 22-03 Plan: Conditional Self-Speech Guard

<objective>
Resolve the 22-02 safety audit: either document that existing mitigations are sufficient, or implement a small deterministic guard before final Chat submission if evidence shows assistant speech can still enter user input.
</objective>

<tasks>

## 1. Read 22-02 Safety Decision

Read `.planning/phases/22-aec-spike-no-headphones-decision/22-SAFETY-AUDIT.md`.

Create `.planning/phases/22-aec-spike-no-headphones-decision/22-SELF-SPEECH-GUARD-DECISION.md` with:

- `existing_mitigations_sufficient`;
- evidence summary;
- implementation decision;
- accepted residual risk;
- tests added.

If `existing_mitigations_sufficient: yes`, do not add production guard code. Instead, add regression tests that prove the current final transcript flow prevents self-submit through pause/queue rules.

If `existing_mitigations_sufficient` is `no` or `unknown`, continue with tasks 2 through 5.

## 2. Implement Deterministic Guard If Needed

Create `apps/renderer/src/audio/self-speech-guard.ts` only when needed.

Inputs should include:

- final transcript candidate;
- active TTS/playback state;
- recent assistant text window if available;
- candidate source mode/provider metadata if available;
- timestamp/latency metadata if available.

Behavior:

- accept when no active/recent TTS risk exists;
- discard exact, near-substring, or high-overlap assistant-output candidates during risky windows;
- accept unrelated user speech if it does not resemble assistant output;
- return redacted reason codes, not full raw transcript diagnostics;
- never call an LLM.

## 3. Wire Guard Before Shared Submit If Needed

Run the guard at the single point where a final voice transcript becomes Chat `text-input`.

Rules:

- local and cloud STT final results must share the same path;
- do not show discarded assistant-like text in Chat or preview;
- preserve one queued final candidate behavior;
- preserve Stop and edit/regenerate behavior;
- preserve no-barge-in rule.

## 4. Add Tests

If guard code is implemented, add tests for:

- assistant replay during active TTS is discarded;
- near-substring/high-overlap assistant candidates are discarded;
- unrelated user speech during active TTS is accepted only when the phase explicitly allows it and it passes guard criteria;
- discarded candidates are not appended to Chat or persisted;
- diagnostics contain reason codes and redacted metadata only;
- no barge-in or stop-turn signal is emitted by the guard.

If no guard code is implemented, add tests proving:

- VAD active-turn pause prevents VAD self-submit;
- queued PTT/final candidate flow does not submit until active TTS/settlement clears;
- Stop/edit recovery remains the user-facing correction path.

## 5. Update Decision Record

Finalize `22-SELF-SPEECH-GUARD-DECISION.md` with:

- exact code/tests changed;
- whether guard module exists;
- why the selected behavior satisfies `AEC-02`;
- limitations to carry into 22-05 UAT.
