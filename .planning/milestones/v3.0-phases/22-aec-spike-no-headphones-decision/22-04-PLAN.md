# 22-04 Plan: Settings No-Headphones Status And VAD Policy

<objective>
Expose truthful no-headphones support status in Settings, keep VAD opt-in and conservative, and provide an explicit Unsafe override without blocking baseline PTT availability.
</objective>

<tasks>

## 1. Extend Renderer Voice Settings Model

Update `apps/renderer/src/state/audio-settings.ts` with renderer-local fields for:

- no-headphones status: `ready`, `limited`, `unsafe`;
- explicit unsafe override boolean;
- optional status diagnostic timestamp/source if needed for Settings display;
- conservative VAD defaults unchanged.

Rules:

- default status must be conservative (`unsafe`) until 22-05 UAT changes it;
- preserve existing local storage migration for PTT shortcut, selected microphone, and VAD settings;
- do not move renderer voice preferences into Electron safe-storage unless a concrete cross-process need appears.

## 2. Add Centralized Copy

Update `apps/renderer/src/lib/copy.ts` with concise copy for:

- no-headphones status heading;
- `Ready`, `Limited`, `Unsafe`;
- recommendation per status;
- Limited/Unsafe recommendation to use PTT + headphones;
- Unsafe override label and warning;
- concise AEC diagnostics/status detail label.

Avoid full UAT notes inline.

## 3. Render Settings-Only Status

Update `VoiceInputSection.tsx` and, if necessary, the parent Settings voice input section so Settings shows:

- current no-headphones status;
- concise recommendation;
- selected microphone warning remains visible;
- AEC diagnostic summary from 22-01 if available;
- Unsafe override control only when status is `unsafe`;
- VAD enablement guarded by explicit override when status is `unsafe`.

Chat should not show no-headphones support status copy.

## 4. Enforce VAD And PTT Policy

Ensure:

- VAD remains disabled by default for all statuses;
- `Ready` never auto-enables VAD;
- `Limited` keeps warning copy and conservative defaults;
- `Unsafe` requires explicit override before VAD can be enabled;
- PTT remains available when STT readiness allows it, regardless of no-headphones status;
- selected microphone loopback warning still appears.

## 5. Add Tests

Add or update tests for:

- default settings normalize to `unsafe`, override false, VAD disabled;
- Ready/Limited/Unsafe labels render in Settings;
- Limited and Unsafe recommend PTT + headphones;
- Unsafe blocks or guards VAD enablement until explicit override;
- Ready does not auto-enable VAD;
- Chat does not render support-status copy;
- PTT availability remains based on STT readiness, not no-headphones status.

</tasks>
