# 22-05 Plan: Structured UAT And Verdict Closure

<objective>
Run and document real no-headphones/AEC evidence, set the shipped Settings status from the verdict, and close all AEC requirements with automated and manual verification notes.
</objective>

<tasks>

## 1. Create UAT Document

Create `.planning/phases/22-aec-spike-no-headphones-decision/22-UAT.md` with:

- purpose and scope;
- explicit no-raw-audio-retention rule;
- environment/device section;
- AEC diagnostics section;
- test matrix;
- result columns: expected, observed, pass/fail, diagnostic reference, notes;
- final verdict: `Ready`, `Limited`, or `Unsafe`;
- limitations and follow-up section.

## 2. Record Environment Metadata

Capture metadata only:

- OS and app build mode;
- selected microphone practical label;
- confirmation selected input is not loopback/system audio;
- speaker/headphone state;
- speaker output volume;
- room noise;
- STT provider and readiness state;
- VAD enabled/disabled, sensitivity, silence timeout;
- no-headphones status before test;
- requested/applied AEC diagnostics from 22-01.

Do not require detailed driver versions unless execution finds a concrete issue where they matter.

## 3. Run Automated Regression

Run the targeted automated checks from plans 22-01 through 22-04:

- AEC diagnostics tests;
- active-TTS/VAD/PTT safety tests;
- self-speech guard or no-guard decision tests;
- Settings status/default tests;
- Chat voice input tests;
- renderer typecheck.

Fix Phase 22 scoped failures before manual UAT.

## 4. Execute Manual UAT Matrix

Run and record:

- physical microphone selected; loopback/system-audio input not selected;
- AEC diagnostics captured for selected input;
- PTT no-headphones while Teto is idle;
- VAD no-headphones while Teto is idle;
- TTS active with no user speech: assistant speech must not submit as user text;
- TTS active with VAD enabled: VAD paused/ignored state visible and no submit;
- PTT during active TTS: final transcript follows queue/safety rules with no assistant self-submit;
- Stop during active Thinking/streaming/TTS cancels and unlocks edit;
- edit/regenerate works after normal and stopped voice turns;
- FunASR local path;
- faster-whisper local path or explicit lower-quality fallback note;
- cloud STT only if credentials and explicit consent are available; otherwise mark skipped with automated consent/redaction coverage;
- Settings status and override behavior.

## 5. Set Final Shipped Status

Based on UAT:

- `Ready` only if strict repeatable evidence passes;
- `Limited` if mostly usable but edge cases remain;
- `Unsafe` if UAT judgment says no-headphones behavior is unusable or too risky.

Update renderer default/copy/tests to match the final verdict. Do not add runtime auto-classification.

## 6. Write Verification Notes

Create `.planning/phases/22-aec-spike-no-headphones-decision/22-VERIFICATION.md` with:

- automated test commands and outcomes;
- UAT verdict;
- requirement coverage table for `AEC-01` through `AEC-04`;
- guard decision summary from 22-03;
- known limitations;
- blocked/skipped checks;
- confirmation that raw audio was not retained.

</tasks>
