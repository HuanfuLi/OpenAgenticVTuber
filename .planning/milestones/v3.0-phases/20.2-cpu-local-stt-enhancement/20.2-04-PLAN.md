# 20.2-04 Plan: Cross-Feature Regression And UAT Evidence

<objective>
Verify Phase 20.2 as one coherent user workflow: final-only local STT, responsive CPU behavior, stop-current-turn, and edit/regenerate recovery for STT typos.
</objective>

<execution_context>
The first three plans touch contracts, renderer capture, sidecar cancellation, TTS queues, renderer audio playback, and conversation persistence. This final plan closes integration gaps, removes stale preview/test assumptions, and records UAT evidence.
</execution_context>

<tasks>

## 1. Run Stale Preview Cleanup Search

Search the codebase for old preview vocabulary:

```powershell
rg "previewing|previewTranscript|previewSequenceId|applyPreviewResult|setTransientPreview|clearTransientPreview|mode: 'preview'|mode: \"preview\"|VOICE_PREVIEW" apps packages sidecar
```

Required outcome:

- no active Chat voice input code references preview transcription;
- any remaining references are archive/planning-only or deliberately unrelated;
- tests are updated to assert final-only behavior.

## 2. Add End-To-End Renderer Regression Coverage

Add or extend renderer tests to cover the full workflow:

- PTT final transcription sends only one final transcription request;
- Send button changes to Stop during streaming/speaking and returns after stop;
- stop clears renderer audio and partial assistant output;
- stopped user message remains editable;
- saved edit dispatches as normal text-input;
- final STT typo can be corrected through the same edit/regenerate path;
- input cannot be edited or sent while a turn is active.

## 3. Add Cross-Tier Sidecar/Electron Regression Coverage

Add or extend tests for:

- sidecar stop handler cancellation and idempotency;
- orchestrator exact memory resync after truncated renderer history;
- TTS manager clear path during stop;
- conversation truncation before selected user message;
- IPC/preload method typing and refresh behavior;
- faster-whisper CPU thread diagnostics where mockable.

## 4. Run Focused Verification Suite

Run:

```powershell
npm run check:contracts
npm --workspace apps/renderer run test -- --run voice-capture voice-input-store ChatVoiceInput ChatStreaming ws-store-audio ConversationHistory
npm --workspace apps/renderer run typecheck
npm --workspace apps/electron-main run test -- --run conversation ipc-conversation ipc-voice-input
npm --workspace apps/electron-main run build
cd sidecar; uv run pytest tests/admin/test_audio_voice_input_endpoint.py tests/admin/test_audio_stt_local.py tests/test_orchestrator_turn.py tests/ws/test_stop_turn.py -q
```

If an existing test file name differs, use the nearest existing focused test names and record the exact command in verification evidence.

## 5. Create UAT And Verification Artifacts

Create `20.2-UAT.md` with manual checks:

- faster-whisper CPU with selected `small` model records and transcribes one final request per PTT utterance;
- no preview text appears while recording;
- repeated utterances avoid visible model reload delay after warmup/cache reuse;
- Stop during Thinking, streaming text, and TTS playback;
- after Stop, edit the user message and regenerate;
- after a completed response, edit an earlier user message and confirm later history truncates;
- voice transcript typo recovery uses edit/regenerate, not undo.

Create or update `20.2-VERIFICATION.md` after execution with:

- automated command results;
- manual/live UAT outcomes;
- residual risks, especially faster-whisper CUDA remaining environment-dependent and `whisper.cpp` Vulkan deferred.
