# Phase 20.2 Context: CPU Local STT Enhancement

## Phase Goal

Voice input should feel responsive on local CPU STT without degrading model quality, and users should be able to recover from STT typos through explicit stop and edit/regenerate flows.

## User Decisions

- Discard live STT preview completely. Voice capture should submit only the final recorded audio for transcription.
- Do not implement grace-window undo, delayed dispatch, or any automatic undo behavior.
- Keep faster-whisper `small`; the small model is already close to the minimum acceptable quality, so latency fixes must not downgrade model quality.
- Let users recover from STT typos by editing a sent user message and regenerating from that point, similar to LM Studio.
- Editing/regeneration should not interfere with the pipeline. Each saved edit dispatches as a normal new request with truncated history.
- Add a Stop feature like Gemini/ChatGPT so users can stop streaming/TTS, then edit the user message without waiting for the current response to complete.
- During streaming or speaking, replace the paper-plane Send button with a red square Stop button.
- Stop is explicit cancellation. It should not delete the user message, should discard partial assistant output, and should stop queued/playing audio.
- Message editing/regeneration is available only when no turn is active.
- Keep note for future work: `whisper.cpp` with Vulkan may be an easier local GPU STT option than faster-whisper CUDA on Windows, but Phase 20.2 focuses on CPU local STT and chat recovery UX.

## Observed Problems

- faster-whisper CUDA failed on Windows because CUDA 12 runtime libraries were missing: `cublas64_12.dll`, `cublasLt64_12.dll`, and related runtime dependencies. This is a packaging/runtime compatibility problem rather than an app-only bug.
- The prior preview transcription path retranscribes accumulated MediaRecorder chunks while recording. On local CPU STT this creates avoidable work, queue pressure, and visible latency.
- Chat PTT can get stuck in `Finalizing` when local STT work is slow or blocked, especially when preview requests compete with final transcription.
- Existing Chat has no way to stop a streaming assistant response or TTS playback once a bad STT transcript has already been submitted.
- Existing conversation history commits only completed turns, so edit/regenerate needs explicit truncation and dispatch behavior rather than mutating completed history in place and duplicating user messages.

## Codebase Findings

- `apps/renderer/src/audio/voice-capture.ts`
  - Starts `MediaRecorder` with `recorder.start(this.previewTimesliceMs)`.
  - `ondataavailable` pushes each chunk and calls `transcribePreview(captureId)`.
  - `transcribePreview` creates a blob from accumulated chunks and calls `window.api.transcribeVoiceInput({ mode: "preview" })`.
  - `finalizeRecording` calls the same bridge with `mode: "final"`.
- `apps/renderer/src/state/voice-input-store.ts`
  - Stores `previewTranscript`, `previewSequenceId`, `setTransientPreview`, and `applyPreviewResult`.
  - `CaptureStatus` includes `previewing`.
- `apps/renderer/src/screens/Chat/VoiceInputControl.tsx`
  - Displays preview and finalizing overlays.
  - Treats `previewing` as pressed/active mic state.
- `apps/renderer/src/screens/Chat/Chat.tsx`
  - `submitFinalText` appends a streaming user message, sends `type: "text-input"`, and passes persisted `activeSession.messages` as history.
  - Send input is disabled while `turnInFlight || banners.llm || !wsOpen`.
  - Voice final candidates use the same `submitFinalText` path.
- `apps/renderer/src/screens/Chat/useStreamingMessages.ts`
  - Holds in-flight user and assistant messages outside persisted history.
  - `markCompletedTurnConsumed` removes streaming messages after persistence commits the completed turn.
  - Needs a cancellation path that keeps the user bubble and removes partial assistant output.
- `apps/renderer/src/ws/audio-player.ts`
  - Queues renderer-side WAV playback but has no public stop/clear function.
- `sidecar/src/sidecar/ws/handlers.py`
  - Handles `text-input`, `shutdown`, and generic `control`.
  - Needs an explicit stop-current-turn control path.
- `sidecar/src/sidecar/orchestrator/orchestrator.py`
  - `_turn_loop` awaits each turn directly from `pending_inputs`.
  - A stop feature needs an active-turn task/cancel hook so an LLM stream and TTS work can be interrupted.
  - `_sync_memory_from_history` currently only replaces memory on session change or when incoming history grows; edit/regenerate after truncation needs an exact history resync path.
- `sidecar/src/sidecar/tts/tts_manager.py`
  - Has `clear()` for canceling TTS tasks and queues.
  - Stop-current-turn should use it and reset ordered queue state.
- `sidecar/src/sidecar/admin/audio.py`
  - Already caches local STT providers with `_STT_PROVIDER_CACHE` and serializes local transcription per provider key.
  - `validate_runtime()` loads local models, so cache reuse exists, but user-facing latency still pays preview work and may not warm at the right user action.
- `sidecar/src/sidecar/stt/providers/faster_whisper_provider.py`
  - Uses `WhisperModel(model_path, device=device, compute_type=compute_type)`.
  - CPU path does not pass bounded `cpu_threads` yet.

## Non-Goals

- No grace-window undo.
- No model downgrade from faster-whisper `small`.
- No wake word, barge-in, or no-headphones/AEC claim.
- No silent fallback between STT providers or from local STT to cloud.
- No attempt to bundle NVIDIA CUDA DLLs in this phase.
- No new GPU STT provider in this phase; keep `whisper.cpp` Vulkan as a future note.
