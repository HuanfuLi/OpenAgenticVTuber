# 20.2-02 Plan: Stop-Current-Turn Protocol And Send/Stop UX

<objective>
Add an explicit Stop action that interrupts streaming and TTS, clears partial assistant output, stops queued/playing audio, and lets the user edit the submitted prompt without waiting for the current turn to finish.
</objective>

<execution_context>
The current Chat pipeline disables input during a turn and has no cancellation path. The user wants the paper-plane Send button replaced by a red square Stop button during streaming/speaking, matching mainstream chat app behavior.

This plan must keep the user message because edit/regenerate depends on it. Editing itself is implemented in plan 20.2-03.
</execution_context>

<tasks>

## 1. Add A Typed Stop-Turn WebSocket Envelope

Update WS contracts.

Required behavior:

- add a message type such as `StopTurnMessage` with `type: "stop-turn"`;
- include it in the Python and generated TypeScript `WSMessage` union;
- add generated schema/codegen updates;
- do not overload existing `control` text values for this primary chat action.

Run:

```powershell
npm run codegen:contracts
npm run check:contracts
```

## 2. Make Sidecar Turns Cancelable

Update `sidecar/src/sidecar/orchestrator/orchestrator.py`.

Required behavior:

- `_turn_loop` should track the currently running turn as an active task;
- expose an async `cancel_active_turn()` method;
- cancellation should interrupt LLM streaming and TTS synthesis/sending where asyncio cancellation can reach;
- call `TTSTaskManager.clear()` on cancellation;
- clear queued pending inputs that were not yet started;
- emit or allow an idempotent `conversation-chain-end` so renderer input unlocks;
- do not append partial assistant text to `_memory`;
- remove or invalidate the uncommitted user message in sidecar memory, or ensure the next request always resyncs exactly from renderer-provided history.

Tests:

- a long-running turn can be canceled by stop;
- cancellation calls the TTS clear path;
- partial assistant output is not committed to sidecar memory;
- queued pending input does not start after stop.

## 3. Add Stop Handler

Update `sidecar/src/sidecar/ws/handlers.py`.

Required behavior:

- register `@on("stop-turn")`;
- if no orchestrator or active turn exists, return without error;
- call orchestrator cancellation;
- never close the WebSocket as part of normal stop behavior;
- log stop events without raw prompt/audio data.

Tests:

- unknown/no-active stop is harmless;
- active stop invokes cancellation and keeps the socket alive.

## 4. Add Renderer Audio Stop

Update `apps/renderer/src/ws/audio-player.ts`.

Required behavior:

- expose `stopAudioPlayback()`;
- clear queued base64 audio;
- pause current audio;
- detach event handlers or make cleanup idempotent;
- revoke active object URLs;
- clear `activeAudio` and `currentAudio`.

Tests:

- queued audio is dropped;
- current audio is paused/stopped;
- repeated stop calls are safe.

## 5. Add Stopped-Turn Streaming State

Update `apps/renderer/src/screens/Chat/useStreamingMessages.ts` and `apps/renderer/src/ws/store.ts`.

Required behavior:

- expose a function such as `stopActiveTurnLocally()` or `stopActiveTurn`;
- when called, keep the pending user bubble;
- remove all assistant bubbles created after the pending user bubble;
- set `inputDisabled: false`;
- set `isSpeaking: false`;
- mark the pending turn as stopped/interrupted so `getCompletedTurnCandidate()` returns null for it;
- make later `conversation-chain-end` from sidecar idempotent and unable to commit a stopped partial turn;
- preserve enough pending user metadata for plan 20.2-03 to edit/regenerate it.

Tests:

- stop removes Thinking and partial assistant text but leaves the user text;
- chain-end after stop does not persist a completed turn;
- banners are not shown for normal user stop.

## 6. Replace Send With Stop During Active Turns

Update `apps/renderer/src/screens/Chat/Chat.tsx` and icon/copy files.

Required behavior:

- while `turnInFlight || isSpeaking`, render the existing `.send` control as a Stop button;
- use a red square icon, adding a local `Square` icon to `apps/renderer/src/lib/icons.tsx` if needed;
- use a clear aria label such as `Stop response`;
- clicking Stop sends the typed stop-turn message when the socket is open;
- clicking Stop immediately performs local stop cleanup and audio stop for responsive UI;
- keep input disabled until local stop cleanup completes, then allow editing in plan 20.2-03;
- Enter should not send a new message while a turn is active.

Styling:

- keep the control in the same layout slot as Send to avoid layout shift;
- make Stop visually red/destructive but not oversized;
- ensure disabled/hover/focus states remain accessible.

</tasks>
