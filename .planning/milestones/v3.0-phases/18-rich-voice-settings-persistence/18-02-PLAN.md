# 18-02 Plan: Voice Output And Voice Input Settings UI

<objective>
Replace the static TTS summary and Voice-in placeholder with truthful voice settings that show provider catalogs, capability labels, selected voice output/preset state, local-first voice input defaults, and explicit cloud-STT consent boundaries.
</objective>

<execution_context>
Use the existing long-scroll Settings style and anchor pills. Do not create a landing page or marketing-style setup page. Settings is an operational surface: compact, scannable, and truthful.

This plan depends on 18-01 typed audio settings/status methods. If Phase 17 already added a voice output section, merge into that section instead of creating competing UI.
</execution_context>

<tasks>

## 1. Add Audio Settings State Hook

Create `apps/renderer/src/state/audio-settings.ts` or an equivalent local hook if the app has a newer state pattern by execution time.

The hook should:
- load stored audio settings;
- load provider catalog and runtime status through `window.api`;
- expose save/update methods that preserve unrelated `StoredConfig` fields;
- expose computed privacy states such as `cloudProviderBlocked`, `missingCredential`, and `missingConsent`;
- never expose raw API keys to display components except as masked input state.

## 2. Add Provider Capability Badges

Create a small reusable component for provider labels.

Badges must support at least:
- Local
- Cloud
- Chinese/English
- Requires API key
- Requires external service
- Requires local model
- Supports test synthesis/transcription
- Supports partial preview, if upstream Phase 19-style metadata exists already

Keep badges visually compact and text-safe at narrow Settings widths.

## 3. Build Voice Output Section

Replace the current `TTS / Voice out` static summary with a real `Voice output` section.

Show:
- active TTS provider;
- available TTS providers with capability labels;
- selected voice preset and reference-audio status from Phase 17;
- health summary and last safe test result if available;
- fallback baseline: Piper remains local fallback/status baseline;
- advanced provider controls behind disclosure if Phase 17 exposes GPT-SoVITS knobs.

Rules:
- Do not promise GPT-SoVITS is installed or bundled.
- If GPT-SoVITS requires an external service, label it as such.
- Reference audio display must use sanitized asset names or redacted paths.
- Provider changes should use the existing save-and-restart flow unless upstream provides safe hot-swap.

## 4. Build Voice Input Section

Replace the placeholder `Voice in` section with a real configuration surface.

Show:
- voice input disabled by default until STT provider implementation lands;
- provider catalog rows for local and cloud STT providers if exposed by upstream;
- local providers first;
- cloud providers disabled until separate API key and explicit consent are present;
- consent copy next to cloud enable/test action: cloud transcription sends microphone audio to the selected provider;
- input mode defaults: push-to-talk as baseline, VAD settings disabled or marked pending until Phase 20 if not implemented;
- model/cache settings summary if upstream exposes it, with no surprise-download promise.

Do not add microphone capture, PTT recording, VAD preview, or final transcript submission in this phase.

## 5. Update Settings Anchors And Copy

Update Settings anchors to include:
- Voice output
- Voice input

Move all new user-visible strings to `apps/renderer/src/lib/copy.ts`.

Remove stale Phase 3 voice placeholder copy for Phase 18 targets. Keep unrelated future placeholders for Skills, Agent, Scheduler, Form factor, and Hotkeys unless roadmap copy has changed.

## 6. Add Renderer Tests

Extend `apps/renderer/tests/Settings.test.tsx` to cover:
- provider capability badges render for TTS and STT providers;
- local providers are shown before cloud STT providers where applicable;
- cloud STT rows are disabled without consent and credential;
- consent and credential enable the test/save path without exposing raw key text;
- reference audio and provider errors render redacted strings;
- Voice input does not submit chat text or create conversation history;
- stale `Coming in milestone-3` voice placeholder is absent.

</tasks>
