# 20.1-03 Plan: Renderer Enablement And Optional Diagnostics Cleanup

<objective>
Update Settings behavior and copy so save/enable flows depend only on required setup, while health/test actions remain available as standalone diagnostics in the relevant panels.
</objective>

<execution_context>
This plan builds on the sidebar IA from 20.1-02, the approved `20.1-UI-SPEC.md`, and the cross-tier contract/runtime cleanup from 20.1-01. It is the main renderer behavior cleanup for the user's explicit request to remove mandatory tests before enabling all configurations.
</execution_context>

<tasks>

## 1. Remove Test-Gate Copy

In `apps/renderer/src/lib/copy.ts`, remove or rewrite copy that says:

- tests are required before activation;
- tests are required before enabling voice input;
- untested providers are not ready;
- save succeeded but enablement was disabled because a test was not passed.

Allowed copy:

- concise required-field explanations for cloud consent, API keys, local model cache, reference audio, provider URL, and paths;
- diagnostic labels such as `Test connection`, `Test synthesis`, `Test selected STT`;
- plain failed diagnostic result text that does not imply permission gating.

## 2. Update GPT-SoVITS Renderer Enablement

In Settings:

- stop computing activation permission from health/test pass state;
- allow activation/save when required GPT-SoVITS fields are complete;
- keep health/test buttons in Voice output as optional diagnostics;
- clear previous GPT-SoVITS diagnostic result when related config changes;
- ensure failed health/test diagnostics do not mutate active provider, active preset, or saved enabled config.

Required blockers remain:

- missing provider selection;
- missing or invalid base URL/service mode;
- missing active preset;
- missing reference audio metadata/path;
- missing provider-specific required fields.

## 3. Update STT/Voice Input Renderer Enablement

In Settings and `VoiceInputSection.tsx`:

- stop blocking `enabled` from prior STT test pass state;
- stop saving `enabled: false` after failed optional tests;
- allow voice input enable/save when selected STT provider required setup is complete;
- preserve hard blockers for missing local model/cache, missing cloud consent/API key, unavailable microphone permission where currently required, and invalid provider config;
- clear previous STT diagnostic result when selected provider/model/cache/credential/language/endpoint changes.

## 4. Keep Optional Tests Local To Relevant Panels

Keep diagnostics where users expect them:

- LLM connection test in Connection.
- GPT-SoVITS health/test synthesis in Voice output.
- STT test transcription in Voice input.

Do not add "recommended before enabling" warnings, checklists, or "not tested yet" banners.

## 5. Update Renderer Tests

Add or update tests that prove:

- no test-gate copy renders;
- untested but otherwise valid GPT-SoVITS config can be saved/enabled;
- untested but otherwise valid STT config can be saved/enabled;
- failed GPT-SoVITS diagnostic test does not change active provider/preset;
- failed STT diagnostic test does not disable voice input;
- changing provider-critical config clears old diagnostic status.

</tasks>
