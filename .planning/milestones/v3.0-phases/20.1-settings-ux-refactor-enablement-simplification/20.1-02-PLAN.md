# 20.1-02 Plan: Settings Sidebar IA And Whole-Screen Rearrangement

<objective>
Replace the current long-scroll Settings screen and anchor pills with a comprehensible sidebar category model that audits the entire screen, not only voice settings.
</objective>

<execution_context>
This plan changes visual organization but should preserve existing setting controls and data flow. It depends on 20.1-01 so the layout does not have to preserve stale test-gate affordances. Read `20.1-UI-SPEC.md` before editing and follow its sidebar layout, typography, spacing, accessibility, and visual-quality constraints.

`Settings.tsx` is large. Keep edits structured: introduce a small category/panel model, move existing sections into panels, and avoid opportunistic rewrites outside Settings.
</execution_context>

<tasks>

## 1. Define The Sidebar Category Model

Create a Settings category list with user-facing labels and concise internal ids.

Required categories:

- Connection
- Voice output
- Voice input
- Avatars
- Integrations
- Conversation
- Memory
- Appearance
- Diagnostics
- About

Panel headings should expose system concepts where useful:

- `Voice output (TTS)`
- `Voice input (STT)`
- `Connection`
- `VTube Studio`
- `Plugins`

Default selected panel must be `Connection`.

## 2. Replace Anchor Pills And Long-Scroll Navigation

Remove the anchor-pills interaction and render:

- a left sidebar with category buttons;
- a right panel for the selected category;
- stable responsive layout that remains usable on narrow widths.

Rules:

- no nested page cards;
- no marketing/hero treatment;
- no explanatory text about how to use Settings;
- stable dimensions for nav buttons and panels so selecting categories does not jump the layout.

## 3. Rearrange Existing Sections

Move existing controls into logical panels:

- Connection: LLM/provider connection and credentials.
- Voice output: Piper/GPT-SoVITS provider, presets, reference audio, optional synthesis/health diagnostics.
- Voice input: STT provider, model cache, microphone/voice capture settings, optional transcription diagnostics.
- Avatars: avatar catalog/import controls.
- Integrations: VTube Studio and plugin runtime controls.
- Conversation: prompt/conversation behavior.
- Memory: existing memory controls only.
- Appearance: UI display settings.
- Diagnostics: logs/status/debug diagnostics.
- About: app metadata.

Preserve existing control behavior while moving sections.

## 4. Hide Deferred Placeholders

Remove the rendering of deferred placeholder cards for Skills, Agent, Scheduler, Form factor, and Hotkeys unless those sections are fully usable.

Do not delete future copy constants if other code still imports them; just stop presenting them in Settings.

## 5. Add Renderer Tests

Update Settings tests for:

- Connection is the default panel.
- Sidebar categories render.
- Selecting Voice output shows TTS/GPT-SoVITS controls.
- Selecting Voice input shows STT/microphone controls.
- Deferred placeholders are not rendered.
- Existing critical controls remain reachable after rearrangement.

</tasks>

<verification>

Run:

```powershell
npm test -- --run apps/renderer/tests/Settings.test.tsx
npm run lint -- --filter renderer
```

If no renderer-only lint target exists, run the nearest existing renderer check and record the command used in verification notes.
</verification>
