# Phase 20.1 Plan 07: Faster-Whisper CUDA Runtime Regression Summary

Closed the faster-whisper CUDA UAT blocker by validating local runtime setup before transcription and adding explicit Settings recovery.

## Changes

- Added `FasterWhisperSTTProvider.validate_runtime()` so model/runtime load failures are surfaced as typed provider health before transcription is attempted.
- Added sidecar `_validate_stt_runtime()` and called it from Settings STT diagnostics and Chat voice-input transcription paths.
- Preserved the no-silent-provider-fallback invariant: CUDA failures remain visible and do not switch STT providers.
- Updated Voice input copy so CPU is clearly the safe default and CUDA dependencies are explicit.
- Added an explicit `Switch runtime to CPU` recovery action after CUDA failures; it changes the draft runtime only and does not save config automatically.
- Added sidecar and renderer regression tests for CUDA validation, diagnostics, redacted runtime details, and explicit CPU recovery.
