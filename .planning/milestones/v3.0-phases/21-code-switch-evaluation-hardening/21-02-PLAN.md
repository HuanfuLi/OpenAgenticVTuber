<objective>
Build a repeatable provider scorecard runner that evaluates every available, ready STT provider against the locked corpus using final transcription requests. The output is redacted evidence in `21-SCORECARD.md`, including local recommendation inputs, cloud skipped/blocked states, runtime metadata, and failure summaries without raw audio or secrets.
</objective>

<implementation_notes>
- Reuse `STTProviderRegistry`, `STTRequest`, readiness/config semantics, local model cache path resolution patterns, and provider error handling. Do not build a second provider stack.
- The runner should be usable in automated tests with fake providers and optionally usable manually for real local/cloud runs.
- Cloud rows must never require credentials during automated tests. When credentials are absent, output `skipped` or `blocked` with redacted reason.
- Record metadata needed to debug quality: provider id, model id, language mode, runtime device, compute type, latency, local audio hash/path token, and failure state.
- Do not persist raw transcripts containing private user audio unless they correspond to the locked generic corpus. Redact provider errors consistently with existing audio diagnostics.
</implementation_notes>

<tasks>
1. Add provider run orchestration.
   - Create `sidecar/src/sidecar/stt/eval/runner.py`.
   - Accept corpus cases, audio manifest entries, and a provider/config list.
   - For each ready provider/case, call provider transcription with `language_mode='auto'` unless the corpus/config explicitly says otherwise.
   - Capture final transcript text, language, latency, diagnostics, runtime metadata, and scoring result.
   - Represent missing audio, missing local model, missing cloud consent, missing cloud key, and provider runtime errors as structured row statuses rather than crashes.

2. Add report rendering.
   - Create `sidecar/src/sidecar/stt/eval/report.py`.
   - Render deterministic markdown tables for provider summary, per-case gates, skipped/blocked cloud status, local recommendation candidates, and notes.
   - Update `21-SCORECARD.md` with current run metadata and placeholders if no live run has been executed.

3. Add recommendation input logic.
   - Compute local-only recommendation candidates from hard-gate pass rate, key-token retention, no-translation failures, semantic pass markers, and latency diagnostics.
   - Exclude `openai` and `groq` from default recommendation decisions even when they score well.
   - Mark recommendation as `unchanged`, `switch-candidate`, or `limited-code-switch` rather than blindly changing defaults.

4. Add automated tests using fake providers.
   - Cover successful local provider scoring, provider error rows, missing audio rows, cloud missing consent/key rows, local recommendation exclusion of cloud, and deterministic markdown output.
   - Cover that the runner consumes final transcript results only and has no renderer preview/input chunk API dependency.
</tasks>
