<objective>
Create the locked Phase 21 bilingual/code-switch evaluation corpus contract and reusable scoring helpers. This plan does not run real providers yet. It defines the evidence unit as final submitted transcripts, protects raw user audio from accidental commits, and gives later plans deterministic pass/fail scoring primitives for semantic/key-token/no-translation checks plus WER/CER diagnostics.
</objective>

<implementation_notes>
- Store corpus metadata in planning artifacts, not raw user voice audio. Use a local ignored directory such as `.planning/eval-audio/phase-21/` for recordings.
- The corpus should contain 20-30 generic utterances covering Chinese-only, English-only, and mixed Chinese/English utterances. Keep examples generic and avoid app, provider, project, or avatar names.
- Add `.gitignore` coverage for the local recording directory while allowing checked-in manifests and scorecards.
- Prefer a small local scorer over adding a heavyweight dependency. If `jiwer` is added, keep it dev/test scoped and justify it in the summary.
- Scoring helpers should accept final transcript strings and corpus cases. They must not depend on renderer preview state or MediaRecorder chunks.
</implementation_notes>

<tasks>
1. Add the corpus documentation and local-audio safety boundary.
   - Create `21-CORPUS.md` with corpus purpose, local recording instructions, privacy rules, audio path/hash policy, case schema, and an initial locked case list.
   - Include fields for case id, expected text, language mix, semantic intent, key tokens, no-translation expectation, and optional local audio filename/hash.
   - Update `.gitignore` so raw local eval audio under `.planning/eval-audio/phase-21/` is ignored.

2. Add corpus parsing and validation helpers.
   - Create `sidecar/src/sidecar/stt/eval/corpus.py`.
   - Define typed dataclasses or Pydantic models for corpus cases and audio manifests.
   - Validate unique case ids, non-empty expected text, at least one Chinese-only case, one English-only case, and one mixed case, and required key tokens for mixed cases.
   - Include helpers to load checked-in corpus metadata without loading raw audio.

3. Add scoring helpers.
   - Create `sidecar/src/sidecar/stt/eval/scoring.py`.
   - Normalize punctuation, capitalization, harmless whitespace, and simplified/traditional-neutral comparisons where practical without erasing language boundaries.
   - Implement hard gates: non-empty transcript, no translation/collapse, semantic pass marker, and key-token retention.
   - Implement diagnostic metrics: CER for all cases and WER-like token distance for space-delimited cases. These diagnostics must not override hard gates.

4. Seed the scorecard artifact.
   - Create `21-SCORECARD.md` with schema, run metadata fields, provider rows, skipped/blocked provider states, and placeholders for Plan 21-02 results.
   - State clearly that all scored transcripts are final submitted transcript evidence, not preview/intermediate chunks.

5. Add tests.
   - Add `sidecar/tests/stt/test_code_switch_eval_scoring.py`.
   - Cover no-translation failures, key-token failures, accepted punctuation/case differences, Chinese CER diagnostics, mixed-language preservation, duplicate corpus id rejection, and ignored raw-audio path policy.
</tasks>
