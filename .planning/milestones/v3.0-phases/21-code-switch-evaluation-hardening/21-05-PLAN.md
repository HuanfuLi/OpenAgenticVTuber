</objective>

<tasks>
1. Add explicit microphone input source settings.
   - Extend `VoiceInputSettings` with a nullable selected input device id and label/safety metadata.
   - Normalize old stored settings without breaking existing PTT/VAD preferences.
   - Add helper logic that flags likely system/loopback devices from labels such as `stereo mix`, `loopback`, `monitor`, `what u hear`, `virtual cable`, `speaker`, `output`, or similar.

2. Use the selected microphone source for PTT and VAD.
   - Pass selected device metadata from `VoiceInputControl` into `VoiceCapture` and `VadController`.
   - Build shared getUserMedia audio constraints that use `deviceId: { exact: selectedId }` when a user-selected microphone exists.
   - Do not silently fall back to the OS default when an explicit selected device is unavailable; surface a clear microphone setup error so the user can choose another input.
   - Keep echo cancellation/noise suppression enabled as hints, but do not present this as Phase 22 AEC/no-headphones support.

3. Add Settings UI for microphone source selection and loopback warnings.
   - In Settings > Voice input, show an input device selector populated from `navigator.mediaDevices.enumerateDevices()` after microphone permission is available.
   - Include a default-system-device option, but visibly warn when the selected/default label appears to be loopback/system audio.
   - Persist the selected microphone metadata with other voice input settings and notify Chat of voice input config changes.
   - Keep tests optional and standalone; do not reintroduce test-before-enable language.

4. Record faster-whisper limitation evidence.
   - Update `21-SCORECARD.md` with the faster-whisper UAT evidence: semantic pass for some English/Chinese cases, poor mixed quality, `三个` normalized to `3`, and `帮我` misrecognized as `Onward`.
   - Keep faster-whisper as `limited-code-switch` and not recommended.
   - Update `21-VERIFICATION.md` so CODE-01/CODE-03 remain open until the same UAT is run with recommended FunASR or another provider passes the hard gates.

5. Add regression tests.
   - Cover selected microphone device id passed to both `VoiceCapture` and `VadController` getUserMedia constraints.
   - Cover selected-device unavailable errors do not fall back silently to default input.
   - Cover Settings device selector persistence and loopback/system-audio warning copy.
   - Cover Chat uses the selected microphone metadata for PTT/VAD capture.
</tasks>
