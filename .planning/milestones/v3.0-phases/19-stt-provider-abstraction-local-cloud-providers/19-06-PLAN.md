# 19-06 Plan: STT Test Audio, Readiness, Event Loop, And Cloud Language Fixes

<objective>
Close the remaining Phase 19 correctness gaps: make Settings STT tests send real WAV data, fix contradictory readiness state, run provider work off the FastAPI event loop, and ensure OpenAI/Groq requests honor explicit language mode while preserving cloud consent and redaction guardrails.
</objective>

<execution_context>
This plan depends on `19-05` so the local provider path/cache semantics are already corrected. It covers the non-download findings from `19-REVIEW.md` and keeps the original Phase 19 boundary: Settings-only provider validation, no Chat voice UX, no VAD/PTT preview, no transcript history, no AEC/no-headphones claims.
</execution_context>

<tasks>

## 1. Make Settings Test Recording Match The WAV Contract

Update `apps/renderer/src/audio/test-recorder.ts` to return valid WAV bytes.

Preferred implementation:

- reuse the existing Phase 20 renderer WAV encoder path from `apps/renderer/src/audio/wav-encoder.ts`;
- request microphone only when the user clicks the Settings test action;
- stop tracks promptly after recording;
- discard raw audio after encoding and test submission;
- preserve the public return shape `audioBase64Wav` if the payload is now truly WAV.

If reuse is not practical, change the cross-tier contract to carry `audio_base64`, `media_type`, and `filename`, then update Electron, sidecar, cloud providers, generated TS, and tests. Prefer the smaller real-WAV fix unless the existing encoder cannot safely serve Settings.

Tests:

- add or update `apps/renderer/tests/test-recorder.test.ts`;
- assert the decoded base64 payload starts with a RIFF/WAVE header;
- assert microphone APIs are not called until the explicit user action;
- assert tracks are stopped on success and failure.

## 2. Fix Readiness State Semantics

Update `STTProviderReadiness` so active readiness has a coherent reason.

Acceptable options:

- add a literal reason such as `ready` or `none`;
- or make `invalidation_reason` nullable and set it to null when `active_allowed=True`.

Required behavior:

- `readiness_from_test_result(...)` must not return `active_allowed=True` with `never_tested`;
- failed health/test states still report specific reasons such as `test_failed`, `health_failed`, `missing_model`, `missing_credential`, or `missing_consent`;
- config fingerprint mismatch still returns `config_changed`;
- generated TypeScript/schema outputs are updated through the existing contracts codegen path;
- renderer/electron tests and fixtures are updated to the new valid state.

## 3. Move STT Work Off The FastAPI Event Loop

Update `sidecar/src/sidecar/admin/audio.py`.

Rules:

- provider construction, model load, and `transcribe(...)` execution must run through `asyncio.to_thread` or a bounded executor;
- the route remains async and returns the same typed success/failure contracts;
- exceptions from provider code still map to `STTProviderError` or typed `external_service_failure` responses;
- local and cloud code paths share the same off-event-loop helper where practical;
- raw audio bytes are not logged or stored.

Cover both:

- `/admin/audio/stt/test`;
- `/admin/audio/voice-input`.

Tests:

- add an admin test that monkeypatches a blocking provider and asserts the route awaits an off-thread helper, or otherwise proves the helper is invoked;
- preserve existing readiness and runtime-failure tests.

## 4. Honor Cloud Language Mode

Update OpenAI and Groq STT providers.

Rules:

- when `STTRequest.language_mode == "auto"`, omit provider language parameters and preserve provider auto-detect behavior;
- when language mode is `zh` or `en`, pass the provider-supported language parameter to transcription requests;
- keep transcription APIs, not translation APIs;
- never send a cloud request unless consent and provider-specific credentials are already present;
- keep redacted diagnostics free of API keys, raw audio, raw transcripts, and full provider exceptions.

Tests:

- OpenAI fake client receives `language="zh"` or `language="en"` when selected;
- Groq fake client receives the corresponding language parameter if the SDK accepts it;
- auto mode omits the language parameter;
- missing consent/credential still blocks before provider construction/network behavior.

## 5. Reconcile UAT And Verification Evidence

Update Phase 19 evidence so it no longer overclaims.

Required changes:

- `19-UAT.md` must distinguish automated mocked/provider-boundary checks from real local-provider acceptance;
- `19-VERIFICATION.md` must no longer say real local download/control passed until `19-05` has real evidence;
- add a manual/live acceptance checklist for at least one local provider and one cloud provider if credentials are available;
- document any unavailable external-provider manual checks as blocked, not passed.

</tasks>
