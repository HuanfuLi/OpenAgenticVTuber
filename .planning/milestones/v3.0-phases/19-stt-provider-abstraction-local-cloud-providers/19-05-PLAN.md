# 19-05 Plan: Real Local STT Model Downloads And Cache Path Binding

<objective>
Close the Phase 19 local-model gaps: implement explicit real local STT model downloads into the app-managed cache, make provider construction use the validated app-managed path, and strengthen local-provider tests so Phase 19 no longer claims real local STT acceptance based only on fakes and non-audio bytes.
</objective>

<execution_context>
This is a Phase 19 gap closure created from `19-REVIEW.md` after Phase 20 live UAT exposed that local voice input cannot proceed without manually placed model files. It supersedes the temporary Phase 20 truthfulness fix only by adding the missing real behavior; it must preserve the Phase 20 rule that placeholder, empty, or marker-only cache entries are not reported as downloaded.

Implementation references:

- Hugging Face Hub `snapshot_download` supports whole-repository downloads, filtering, custom `local_dir`, dry-run metadata, and cache control: https://huggingface.co/docs/huggingface_hub/main/guides/download
- faster-whisper uses `WhisperModel(...)` with model identifiers and can auto-download named models from Hugging Face; this plan must avoid surprise downloads during provider load by passing a verified local path after explicit download: https://github.com/SYSTRAN/faster-whisper/blob/master/README.md
- FunASR `AutoModel(model=...)` accepts either a model repository id, which can auto-download, or a path to an already downloaded local model. Provider load must use the local path once the app-managed cache is marked downloaded: https://github.com/modelscope/FunASR/blob/main/docs/tutorial/README.md
</execution_context>

<tasks>

## 1. Add A Provider-Specific Download Layer

Create `sidecar/src/sidecar/stt/model_downloader.py`.

Responsibilities:

- expose a narrow function such as `download_local_stt_model(provider_id, model_id, destination)`;
- run only after the user invokes the download endpoint;
- never import FunASR, ModelScope, Hugging Face, CTranslate2, faster-whisper, or torch at sidecar boot;
- write into a temporary app-managed directory first, then atomically mark the final cache path downloaded only after usable model contents exist;
- leave partial/incomplete state on interruption or failure;
- return typed summaries without raw tracebacks or full user-private paths.

Provider behavior:

- FunASR/SenseVoiceSmall: use the provider-supported ModelScope/FunASR download path where available, or an explicit ModelScope snapshot helper if needed, but ensure the final usable model lives under `STTModelCache.model_path("funasr", "iic/SenseVoiceSmall")`.
- faster-whisper: use Hugging Face Hub `snapshot_download` with `local_dir` pointed at the app-managed cache path for the selected CTranslate2-compatible model, initially `small`.
- do not pass remote model ids to provider constructors as a way to download models.

Dependency behavior:

- add only the minimum download helper dependencies needed in `sidecar/pyproject.toml`;
- keep heavyweight provider imports lazy and inside explicit download or provider test paths;
- missing optional download dependencies should produce actionable `unavailable` or `misconfigured` results.

## 2. Replace The Manual-Setup Stub With Real Download Semantics

Update `STTModelCache` and `/admin/audio/stt/models/download`.

Rules:

- the endpoint performs, or starts and tracks, a real explicit download for supported local providers;
- `ok: true` plus `status: downloaded` is returned only when non-empty usable model contents are present at the app-managed cache path;
- `operation_pending` is allowed for in-progress downloads if a background job is used;
- failed downloads return `ok: false` with `status: incomplete` or `not_downloaded`, never `downloaded`;
- existing placeholder-only or marker-only directories remain `incomplete`;
- remove remains restricted to paths under the app-managed cache root.

If a background job is used, make the existing model catalog endpoint sufficient for polling final state so renderer changes can stay small.

## 3. Resolve And Pass The Effective Local Model Path

Update the admin/provider boundary so cache validation and provider loading use the same path.

Implementation outline:

- add a helper such as `_resolve_local_model_path(request, config)` that returns a validated `Path | None`;
- if `local_model_path_override` is set, verify it contains usable model contents and keep using it;
- otherwise, use `STTModelCache.model_path(provider_id, model_id)` when catalog status is `downloaded`;
- before `STTProviderRegistry().build_provider(...)`, clone/update the config so `local_model_path_override` is the resolved path string for local providers;
- keep readiness blocked when no valid resolved path exists;
- keep cloud providers unaffected.

Provider updates:

- `FunASRSTTProvider.ensure_loaded()` must load `AutoModel(model=<resolved local path>)` for app-managed cache models.
- `FasterWhisperSTTProvider.ensure_loaded()` must load `WhisperModel(<resolved local path>, device="cpu", compute_type="int8")`.
- provider tests must assert the fake constructor receives the resolved local path, not just `"iic/SenseVoiceSmall"` or `"small"`.

## 4. Strengthen Local Audio Test Fixtures

Replace endpoint-level local STT tests that use `base64.b64encode(b"wav")` with a tiny valid WAV fixture.

Rules:

- add a test helper that generates a short valid RIFF/WAV payload in memory;
- keep provider-library behavior mocked so CI does not require downloaded real models;
- assert that invalid/non-audio payloads fail before readiness is marked active where practical;
- keep one focused test proving fake providers can still mark readiness after receiving a valid WAV payload and a real cache path.

Manual acceptance evidence must be explicit:

- add a Phase 19 UAT/manual follow-up item requiring one real local provider download and Settings test on a developer machine;
- do not mark real local STT acceptance complete until that manual/live check passes or is explicitly waived.

## 5. Preserve Phase 20 Readiness And Truthful Copy

After real download behavior lands, verify the Phase 20 surfaces still behave correctly:

- Settings model-cache action no longer says automatic download unavailable when the download succeeds;
- failed/missing dependency/network states remain truthful and actionable;
- Chat voice input remains blocked until model status is downloaded and Settings STT test readiness is active;
- VAD copy remains model-free and separate from STT model cache copy.
