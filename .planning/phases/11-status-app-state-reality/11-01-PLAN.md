Land Phase 11: make the status bar, app store, and setup-derived app state truthful. Replace mock status and mock persistence in normal production paths with existing Electron safeStorage, Electron-store preferences, real sidecar lifecycle events, and a small real sidecar VTS status endpoint.

This plan does not clean every Settings placeholder and does not enforce the full mock boundary. Phase 12 owns Settings reality copy/content. Phase 13 owns broad fixture/mock isolation.

- `apps/renderer/tests/Settings.test.tsx` (extend)

Behavior:

1. Add a StatusIcon test that renders with a mocked `window.api.getStoredConfig()` returning LM Studio plus `modelName: "teto-test-model"`. Open the popover and assert:
   - provider/model detail includes the stored model and provider label
   - it does not include `qwen2.5`
   - it does not include `last reply`
2. Add a StatusIcon interaction test that clicks the retry/refresh button and asserts no mock timeout status is written. If the final UI uses real restart/refresh, assert the real `window.api` method is called. If the control is disabled, assert the disabled state and accurate tooltip/copy.
3. Add AppStore tests for sidecar lifecycle:
   - initial `getReadyUrl()` null yields sidecar amber/red with truthful detail
   - `onSidecarReady(url)` sets sidecar green with the real URL
   - `onSidecarCrash({willRespawn: true})` sets amber/restarting
   - `onSidecarCrash({willRespawn: false})` sets red/permanent failure detail
4. Extend Settings connection tests so its re-test button no longer sets `qwen2.5` or `last reply 423ms`. Prefer asserting the same real refresh/restart call used by StatusIcon, or disabled truthful copy.
5. Add sidecar admin tests for `/admin/vts-status` by constructing a FastAPI app/client state or calling the endpoint helper directly:
   - authenticated/connected state when a completed handshake task exists
   - pending state while handshake task is not done
   - unavailable state when startup error or writer is absent
   - window detection detail when `find_vts_hwnd(force_reprobe=True)` returns None

Expected first run: tests fail because production code still imports `mockStatus`, `mockSafeStorage`, and no VTS status endpoint exists.

# Task 2: Add Real Preference And Status Bridge APIs

Files:

- `apps/electron-main/src/window-store.ts`
- `apps/electron-main/src/ipc.ts`
- `apps/electron-main/preload/index.ts`
- `apps/electron-main/preload/index.d.ts`
- `sidecar/src/sidecar/admin/status.py` (create)
- `sidecar/src/sidecar/admin/__init__.py`
- `sidecar/src/sidecar/ws/server.py`

Implementation:

1. Extend Electron-store schema with non-secret app preferences:
   - existing `chrome` fields remain the source for logs drawer enabled/collapsed/height
   - add `themePreference` with the renderer theme preference shape, or an equivalent JSON object validated at the renderer edge
2. Add IPC/preload methods:
   - `getChromeState(): Promise<ChromeState>`
   - `saveChromeState(patch: Partial<ChromeState>): Promise<ChromeState>`
   - `getThemePreference(): Promise<ThemePrefs | null>`
   - `saveThemePreference(prefs: ThemePrefs): Promise<void>`
   - `getVtsStatus(): Promise<VtsStatus>`
   - `restartSidecar(): Promise<void>` only if the retry button stays actionable
3. Implement `getVtsStatus` in Electron main by fetching `GET /admin/vts-status` from `getSidecarHttpUrl()`. If the sidecar is not ready, return a typed unavailable response instead of throwing into the renderer.
4. Add `sidecar/src/sidecar/admin/status.py`:
   - define a small response shape, for example `{state, detail, authenticated, windowDetected}`
   - read only `request.app.state`; do not instantiate `PyvtsSafeWriter`
   - use `find_vts_hwnd(force_reprobe=True)` for window-presence detail
   - distinguish `authenticated`, `auth_pending`, `not_authenticated`, `sidecar_unconfigured`, and `vts_window_not_found` as much as current state supports
5. Mount the router in `sidecar/src/sidecar/ws/server.py` beside existing admin routers.

Guardrails:

- Do not create a second VTS client.
- Do not persist secrets in Electron-store; provider config and API key stay in safeStorage.
- Do not widen sidecar network binding; all status fetches remain localhost through the existing sidecar URL.
