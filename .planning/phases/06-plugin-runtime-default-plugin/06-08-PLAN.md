<objective>
Close the diagnosed Phase 6 UAT gap where `[joy]` was treated as a valid production expression/action target even though the active imported Teto avatar catalog does not contain a `joy` variant/expression.

Purpose: The user clarified that production variant/expression codes must come strictly from the active imported avatar's own model catalog. For the current Teto avatar, `avatars/重音テト/_avatar_overrides.yaml` lists variants such as `heart-eye`, `star-eye`, `chibi`, and `cry`, but no `joy`. Therefore `[joy]` is obsolete M1/OLVT vocabulary for this avatar, not a Phase 6 UAT success criterion.

Output: Tests and implementation that stop advertising or accepting `[joy]` in the Phase 6 default-plugin production path, plus corrected UAT/verification language. Full `{variant}` / `<event>` parser dispatch remains Phase 7 work.
</objective>

- Do NOT switch to live2d-py or introduce a new renderer.

- Do NOT add exp3 activation or any plugin-side VTS call.
- Do NOT modify the active Teto `_avatar_overrides.yaml` to add `joy`; the absence of `joy` is the point of the regression.

**Decision fidelity:** This plan intentionally supersedes earlier Phase 6 plan language that preserved the OLVT 8-emotion vocabulary.
