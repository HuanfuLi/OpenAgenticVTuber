# 16-01 Plan: Audio Contracts and Config Migration

<objective>
Create the shared audio-provider contract and app-config migration foundation for v3.0 without changing runtime TTS behavior yet.

This plan owns cross-language contract additions and Electron safeStorage migration. Later plans consume these types to move Piper behind a provider shell and expose health checks.
</objective>

  <name>Task 1: Add shared audio provider contracts</name>
  <files>
    - packages/contracts/py/contracts/audio_provider.py
    - packages/contracts/py/contracts/__init__.py
    - packages/contracts/scripts/codegen.py
    - packages/contracts/tests/test_codegen.py
  </files>
  <behavior>
    - Contract code defines provider ids for at least `piper`, `gpt_sovits`, `funasr`, `faster_whisper`, `openai`, and `groq` without implementing future providers.
    - Contract code defines health states: `ok`, `unavailable`, `missing_credential`, `external_service_failure`, `timeout`, `misconfigured`.
    - Contract code defines a minimal v3.0 app audio config with TTS defaulting to Piper/en_US-amy-medium and STT disabled.
    - TTS config includes synthesis timeout fields needed by provider execution, but does not expose GPT-SoVITS settings yet except as future-compatible optional objects if needed.
    - Health result includes provider id, state, summary/detail, retryable flag, optional latency, and redacted diagnostic detail.
  </behavior>
  <action>
    Add `audio_provider.py` Pydantic models. Suggested shapes:
    - `AudioProviderKind = Literal["tts", "stt"]`
    - `AudioProviderId = Literal["piper", "gpt_sovits", "funasr", "faster_whisper", "openai", "groq"]`
    - `AudioHealthState = Literal["ok", "unavailable", "missing_credential", "external_service_failure", "timeout", "misconfigured"]`
    - `AudioProviderHealth`
    - `PiperTTSConfig`
    - `TTSProviderConfig`
    - `STTProviderConfig`
    - `AudioConfig`

    Keep defaults explicit and serializable. Avoid secrets in this first contract; cloud STT credentials are later Phase 18/19 work.

    Export the new types from `contracts/__init__.py`, register `audio-provider` in codegen targets, and add tests for defaults and invalid health states/provider ids.
  </action>
  <verify>
    <automated>python -m pytest packages/contracts/tests/test_codegen.py -q</automated>
  </verify>
  <done>
    Audio provider/config/health contracts exist as Pydantic source of truth and have focused unit coverage.
  </done>
</task>
