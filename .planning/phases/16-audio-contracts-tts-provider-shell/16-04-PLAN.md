<task type="auto" tdd="true">
  <name>Task 1: Add renderer audio payload playback helper</name>
  <files>
    - apps/renderer/src/ws/audio-player.ts
    - apps/renderer/tests/ws-audio-player.test.ts
  </files>
  <behavior>
    - Given a base64 WAV string from `AudioPayloadMessage.audio`, renderer playback creates an audio element/object URL and calls `play()`.
    - Object URLs are revoked after playback ends or errors.
    - `null`, empty, or invalid audio payloads do not throw into the WS dispatcher.
    - Playback failures are caught and logged with `console.warn` only; chat streaming must continue.
  </behavior>
  <action>
    Create a small helper such as `playAudioPayload(audioB64: string | null): void`.

    Use browser-native APIs already available in Electron renderer:
    - `atob()` to decode base64;
    - `Blob` with `audio/wav`;
    - `URL.createObjectURL()`;
    - `new Audio(url).play()`.

    Keep the helper independent of React and app store state so it is easy to test by stubbing `Audio` and `URL`.
  </action>
  <verify>
    <automated>npm --workspace apps/renderer run test -- --run ws-audio-player.test.ts</automated>
  </verify>
  <done>
    Renderer can play the WAV payload already emitted by the sidecar.
  </done>
</task>
