Update `orchestrator.py` to import `ActionCode`, `Dispatch`, `EventFire`, and
`VariantToggle`. Add constructor args `plugin_adapter=None`,
`variant_state_manager=None`, `discrete_dispatcher=None`, and
`event_completion_tracker=None`. Preserve Phase 6 sentence forwarding through
`plugin_adapter.enqueue_sentence(sentence_text)`.

Replace every `sentence_output.actions` use with
`sentence_output.dispatches`.

Add `async def _route_dispatches(self, dispatches: list[Dispatch]) -> None`
with this behavior:
- for `ActionCode`, call `self.plugin_adapter.enqueue_action_code(dispatch)` if
  a plugin adapter exists. Log `[DISPATCH] kind=action name=<name>` on success.
  If there is no adapter or enqueue returns `False`, log
  `[DISPATCH-DROP] kind=action name=<name> reason=plugin-adapter-unavailable`
  or `reason=action-code-queue-full`.
- for `VariantToggle`, await `self.variant_state_manager.apply(dispatch)` when
  present, then log `[DISPATCH] kind=variant name=<name> hotkey_id=<id>`.
- for `EventFire`, if `hotkey_id` is empty log
  `[DISPATCH-DROP] kind=event name=<name> reason=missing-hotkey-id` and skip.
  Otherwise await `self.discrete_dispatcher.fire(dispatch.hotkey_id,
  name=dispatch.name)` when present, call
  `self.event_completion_tracker.track(dispatch)` when present, and log
  `[DISPATCH] kind=event name=<name> hotkey_id=<id> duration_ms=<ms>`.

Call `_route_dispatches(sentence_output.dispatches)` exactly once per sentence
after audio/stub emission succeeds.

Update `tts_manager.py` and `audio_payload_helpers.py` to rename action fields,
parameters, and locals to `dispatches`, type them as `list[Dispatch]`, and
construct `AudioPayloadMessage(dispatches=dispatches, ...)`.
