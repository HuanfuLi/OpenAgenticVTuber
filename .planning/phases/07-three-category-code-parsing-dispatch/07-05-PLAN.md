    - Test 2: applying `VariantToggle("hold-mic", "hk-a")` fires `hk-a` and sets current state.
    - Test 3: applying the same `hk-a` again is a no-op and does not fire.
    - Test 4: applying `hk-b` after `hk-a` fires `hk-a` first, then `hk-b`.
    - Test 5: tracking `EventFire(duration_ms=2833)` sleeps/logs completion after `3.833` seconds.
    - Test 6: tracking missing, zero, or greater-than-10000 duration sleeps/logs completion after exactly `10.0` seconds, not `11.0` seconds.
    - Test 7: tracking the same hotkey twice keeps both tasks active while `in_flight_set()` returns that hotkey once.
  </behavior>
  <action>
Create `VariantStateManager` with constructor
`(dispatcher: DiscreteDispatcher, reset_hotkey_id: str | None = None,
reset_hotkey_name: str = "RemoveAllExpressions")`, `current_hotkey_id`,
`reset_to_baseline()`, and `apply(toggle: VariantToggle)`.

`apply` must no-op when the new hotkey ID equals the current hotkey ID. When a
different current hotkey exists, it must fire the old hotkey first, then fire
the new hotkey, then update shadow state. It must never import or call `pyvts`
directly; all VTS writes go through `DiscreteDispatcher`.

Create `EventCompletionTracker` with constants `BLEND_PAD_MS = 1000` and
`CEILING_MS = 10000`, an internal `dict[str, list[asyncio.Task]]`, `track(event:
EventFire)`, `in_flight_set()`, and async `close()`. `track` creates a task and
then logs `[EVENT-COMPLETE] hotkey_id=<id> name=<name>` after the computed delay.
The delay contract is branch-specific: when `0 < event.duration_ms <= CEILING_MS`,
sleep `(event.duration_ms + BLEND_PAD_MS) / 1000.0`; when `event.duration_ms` is
missing, zero, negative, non-numeric, or greater than `CEILING_MS`, sleep exactly
`CEILING_MS / 1000.0`. Do not add `BLEND_PAD_MS` to fallback durations.

Export both classes from `sidecar/src/sidecar/vts/__init__.py`.

Tests should use a fake dispatcher recording `(hotkey_id, name, force)` tuples
and monkeypatch `asyncio.sleep` inside the tracker module.
  </action>
