# 21-04 Plan: Live Chat UAT And Final Evidence Closure

<objective>
Close Phase 21 with final scorecard evidence, focused regression coverage, and a short live Chat voice UAT proving Chinese, English, and mixed utterances work without manual language switching and without translation.
</objective>

<execution_context>
This plan depends on the scorecard runner, badge/recommendation updates, and any targeted hardening from 21-03. It should not start until Phase 19/20 voice input can be exercised or explicitly documented as environment-blocked.
</execution_context>

<tasks>

## 1. Refresh Automated Scorecard

Run the Phase 21 eval runner with the latest provider code and available corpus audio.

Update `21-SCORECARD.md` with:

- exact command;
- date/time;
- provider ids/models/options;
- corpus/audio metadata;
- per-provider summary;
- local default recommendation;
- Settings badge outcomes;
- blocked provider reasons where applicable.

If user-recorded audio is not available, mark the scorecard as blocked for live evidence and do not claim Phase 21 complete.

## 2. Run Focused Regression Tests

Run all tests added in 21-01 through 21-03 plus relevant Phase 20 Chat voice tests.

Fix failures inside Phase 21 scope only:

- eval harness;
- scorecard aggregation;
- provider output hardening;
- Settings badges/copy;
- tests that assert no translation/no runtime policing/no cloud default.

Do not modify unrelated Phase 19/20 voice capture behavior unless the regression directly breaks Phase 21 evidence.

## 3. Create Live UAT Script

Create `.planning/phases/21-code-switch-evaluation-hardening/21-UAT.md`.

Include short manual checks for:

- provider selected/enabled from Phase 19 readiness;
- no manual language mode switching before tests;
- pure Chinese utterance reaches Chat final transcript unchanged;
- pure English utterance reaches Chat final transcript unchanged;
- mixed Chinese/English utterance reaches Chat final transcript unchanged;
- key tokens survive in at least one mixed utterance;
- translated/collapsed-language behavior is absent;
- Settings badges match scorecard-backed recommendation;
- cloud providers are not shown as default.

Keep the script short enough to run after automated scorecard evaluation.

## 4. Execute Live Chat Voice UAT

With the app running and STT provider ready:

- perform each `21-UAT.md` check;
- record pass/fail, provider id/model, microphone condition, date, and any blocked reason;
- confirm final transcripts enter the normal Chat path and conversation history semantics are unchanged;
- confirm preview text is not mistaken for scorecard evidence.

If live UAT fails, document the failure and create a gap for `$gsd-execute-phase 21 --gaps-only` rather than marking Phase 21 complete.

## 5. Write Verification Notes

Create `.planning/phases/21-code-switch-evaluation-hardening/21-VERIFICATION-NOTES.md`.

Summarize:

- `CODE-01` evidence from live Chat UAT;
- `CODE-02` corpus and scorecard artifact status;
- `CODE-03` recommendation evidence and no-translation/key-token outcomes;
- `CODE-04` Settings badge/copy behavior;
- any environment-blocked provider tests;
- any accepted limitations.

## 6. Final Coverage Check

Manually confirm:

- all `CODE-01` through `CODE-04` are covered by scorecard, tests, or UAT;
- all Phase 21 decisions `D-01` through `D-20` appear in artifacts or verification notes;
- no raw user voice audio is staged;
- no cloud provider is default;
- no runtime transcript language policing was added.

</tasks>
