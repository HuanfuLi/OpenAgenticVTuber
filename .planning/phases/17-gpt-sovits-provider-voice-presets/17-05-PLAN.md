# 17-05 Plan: App-Managed GPT-SoVITS Launch Lifecycle

<objective>
Add optional app-managed GPT-SoVITS launch/stop/restart controls for a user-provided command while guaranteeing the app never kills external servers or mutates user environments.

Purpose: Users who want convenience can launch their own server command from the app, while external-server users remain in full control of their process.
Output: Electron main process manager, IPC/preload methods, copy strings, and tests for app-owned-only termination.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| renderer→Electron process control IPC | User clicks start/stop/restart and Electron launches/kills a local process. |
| user command→OS process | User-provided command/cwd is executed by the app. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-17-10 | E | `gpt-sovits-process.ts` command execution | mitigate | Execute only user-supplied command/cwd; no templates, no env mutation, no dependency install per D-02. |
| T-17-11 | D | stop/restart controls | mitigate | Track child process handle/PID and stop only that app-owned process tree; never kill external servers per D-04. |
| T-17-12 | I | process stdout/stderr | mitigate | Capture diagnostics for logs with bounded/redacted output; main UI uses concise copy. |
</threat_model>
