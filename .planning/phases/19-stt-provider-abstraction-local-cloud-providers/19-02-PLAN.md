# 19-02 Plan: Local STT Providers And Model Cache Controls

<objective>
Implement local STT providers for FunASR/SenseVoiceSmall and faster-whisper behind the 19-01 STT interface, with explicit model download/remove/status, lazy imports, non-blocking transcription, and readiness integration.
</objective>

<execution_context>
This plan depends on 19-01 contracts, registry, model cache, readiness helpers, and admin endpoint skeletons.

Keep local provider dependencies lazy. Adding package dependencies to `sidecar/pyproject.toml` is allowed, but importing them at sidecar boot is not. Tests should use fakes/mocks; real model download and transcription can remain manual verification because CI must not require global model caches or network model downloads.
</execution_context>

<tasks>

## 1. Add Local Provider Dependencies With Lazy Import Discipline

Update `sidecar/pyproject.toml` for local STT dependencies if the project does not already have a dependency group/extras pattern.

Candidate dependencies:

- `funasr`
- `modelscope` and/or `huggingface_hub`
- `faster-whisper`
- `soundfile` only if needed for decoding Settings test WAVs

Do not add CUDA-specific dependencies. Do not bundle model files. If torch/torchaudio pinning is needed, choose a Python 3.12-compatible CPU path and document any platform caveat in code comments or tests.

## 2. Implement FunASR/SenseVoiceSmall Adapter

Create `sidecar/src/sidecar/stt/providers/funasr_provider.py`.

Rules:

- no top-level import of `funasr`, `torch`, `torchaudio`, `modelscope`, or `huggingface_hub`;
- lazily import and construct `AutoModel` inside explicit model load/test/transcribe flow;
- default model id is SenseVoiceSmall (`iic/SenseVoiceSmall` or the model id chosen by the contract/catalog);
- default language mode is auto;
- CPU is the safe default; GPU/device options remain minimal and internal unless needed for basic operation;
- strip provider-specific tags only if they are not user speech, while preserving transcript language/content;
- return `STTProviderError` states for missing model, missing dependency, timeout, no speech, unsupported audio, and internal errors.

The adapter should accept a local cache path after explicit download. It should not trigger provider library auto-download by passing only a remote model name when the model is not marked downloaded.

## 3. Implement faster-whisper Adapter

Create `sidecar/src/sidecar/stt/providers/faster_whisper_provider.py`.

Rules:

- no top-level import of `faster_whisper` or CTranslate2-dependent modules;
- lazy construct `WhisperModel`;
- default to CPU/int8 or equivalent low-risk local default;
- default model size should be modest (`small` unless existing catalog chooses another safe default);
- language auto-detect by default;
- set options conservatively to avoid cross-utterance carryover where supported, such as `condition_on_previous_text=False`;
- map provider errors to the same health/error taxonomy.

## 4. Implement Real Local Model Status, Download, And Remove

Extend `sidecar/src/sidecar/stt/model_cache.py` and admin endpoints so Settings can:

- show cache root path;
- show FunASR/SenseVoiceSmall and faster-whisper model statuses;
- download a selected local model only after explicit user action;
- remove app-managed local model files;
- refuse removal of non-app-managed external paths.

Use provider-supported download mechanisms where practical, but ensure downloads land in or are mirrored/registered under the app-managed cache root. If a provider library cannot download to the chosen path safely, implement status as "manual path required" rather than surprise-downloading to a global cache.

Add timeout/cancellation-safe behavior where the existing admin request style allows it. At minimum, return clear failure status and leave partial downloads marked incomplete.

## 5. Wire Local Providers Into Registry And Admin Endpoints

Update the 19-01 registry/admin endpoint skeletons:

- local provider health reports missing dependency, missing model, not loaded, ready, timeout, and internal error states;
- test transcription accepts the Settings WAV payload, normalizes it for provider input, runs provider work off the event loop, and discards raw audio immediately after transcription;
- successful local test updates readiness with transcript-present state;
- runtime failure invalidates readiness and does not auto-fallback to another provider.

Keep admin responses redacted and bounded.

## 6. Add Local Provider Tests

Add tests using monkeypatch fakes for provider libraries:

- FunASR adapter imports `funasr` only during explicit load/test, not module import.
- faster-whisper adapter imports `faster_whisper` only during explicit load/test, not module import.
- missing dependency maps to unavailable/misconfigured health without sidecar crash.
- missing local model blocks test and enable.
- fake successful transcription returns non-empty text and marks readiness allowed.
- empty transcript does not mark readiness allowed.
- provider runtime error marks provider unhealthy/unready and never falls back.
- model cache remove cannot delete paths outside app-managed cache.

</tasks>
