# 19-04 Plan: Settings STT Test Recorder, Cache UI, And Enablement Gate

<objective>
Expose the completed Phase 19 STT provider layer in Settings: provider catalog, local model cache controls, cloud consent/credentials, short manual test recording, test result diagnostics, and readiness-gated provider enablement for future voice input.
</objective>

<execution_context>
This plan depends on local provider adapters from 19-02 and cloud adapters from 19-03.

Do not build Phase 20 voice chat UX here. The recorder is a Settings validation tool only. It must not submit chat messages, create transcript preview streams, add VAD/PTT controls, or alter conversation history.
</execution_context>

<tasks>

## 1. Add Electron IPC And Preload STT Methods

Extend `apps/electron-main/src/ipc.ts`, `apps/electron-main/preload/index.ts`, and `apps/electron-main/preload/index.d.ts` with typed methods for:

- listing STT provider catalog/status;
- reading local STT model/cache status;
- downloading a local STT model;
- removing an app-managed local STT model;
- running STT health check;
- running STT test transcription with a short WAV payload;
- enabling/disabling the selected provider only when readiness allows it;
- saving cloud provider credentials/consent in the existing encrypted config path.

Proxy sidecar admin endpoints through Electron as existing audio status methods do. Renderer must not call sidecar directly.

IPC rules:

- do not log raw audio payloads;
- do not log API keys;
- handle sidecar-not-ready as a typed unavailable state;
- block cloud test/enable in Electron too when consent or credential is missing.

## 2. Add Or Extend Renderer Audio Settings State

Create or extend `apps/renderer/src/state/audio-settings.ts`.

The hook/state module should:

- load stored audio config;
- load STT provider catalog/cache/readiness status;
- expose update/save helpers that preserve unrelated stored config fields;
- compute blocked reasons such as missing model, missing consent, missing credential, health not checked, test not passed, and readiness invalidated;
- expose masked cloud credential editing state without rendering raw saved keys.

## 3. Build A Settings-Only Test Recorder

Add `apps/renderer/src/audio/test-recorder.ts`.

Behavior:

- request microphone permission only when the user clicks record/test;
- record a short manual sample with a fixed max duration, such as 8-10 seconds;
- encode mono WAV in memory, preferably normalized to 16 kHz when practical;
- return base64 WAV plus duration to the Settings test call;
- stop tracks and release microphone promptly after recording;
- discard raw audio after the test promise settles;
- expose errors for permission denied, no input, too short, and encoding failure.

Do not create `VoiceInputController`, `/voice/ws`, PTT hotkeys, VAD state, or chat submission in this phase.

## 4. Build Voice Input Settings Section

Create `apps/renderer/src/screens/Settings/VoiceInputSection.tsx` or extend the existing Settings section if Phase 18 already created it.

Show:

- local providers before cloud providers;
- FunASR/SenseVoiceSmall marked as recommended local default for Chinese/English/code-switching;
- faster-whisper marked as local fallback;
- OpenAI and Groq marked as cloud, requiring API key and consent;
- provider capability labels from catalog;
- local model cache path, status, size label, download, and remove controls;
- cloud consent copy as one-time setup state;
- health status and readiness status;
- short manual test recording action;
- latest test result transcript and diagnostics: provider, duration, latency, model/cache state, and redacted error;
- enable control blocked until readiness allows it.

Rules:

- no broad advanced CPU/GPU/quantization panel unless a local provider cannot work without one;
- no chat submit button;
- no preview stream;
- no VAD/PTT controls beyond disabled/future labels if Phase 18 already has placeholders.

## 5. Update Settings Anchors And Copy

Update `Settings.tsx` anchors and `COPY` strings so Voice Input is a first-class Settings section.

Use concise operational copy, not a landing page. Cloud copy should explain once that cloud transcription sends microphone audio to the selected provider.

Do not repeat per-action cloud confirmations after consent is saved.

## 6. Persist Readiness And Invalidate On Relevant Changes

When the user changes provider id, local model id/path, cache root, cloud credential-present state, cloud endpoint/model id, consent flag, or language mode:

- clear or invalidate readiness in stored audio config;
- require health plus successful test again before enablement;
- preserve unrelated TTS, LLM, plugin, avatar, and conversation config.

When a provider runtime failure is reported by sidecar status:

- show unhealthy/disabled state;
- prevent future voice-input enablement until re-test passes;
- do not switch to another provider automatically.

## 7. Add Renderer And Electron Tests

Extend `apps/renderer/tests/Settings.test.tsx` and add focused recorder tests.

Cover:

- provider catalog renders local providers before cloud providers;
- FunASR is visibly recommended and faster-whisper is visible as local fallback;
- local model cache path/status/download/remove controls render;
- cloud providers are blocked without credential and consent;
- saved consent plus masked credential enables cloud test action;
- test recorder calls microphone APIs only after explicit user action;
- test result transcript appears only in the immediate result panel;
- no conversation submission or history call is made by the test recorder;
- readiness blocks enable until health and successful test pass;
- config changes invalidate readiness.

Extend Electron tests for safe config preservation and IPC blocked states where practical.

## 8. Final Cross-Provider Regression

After local and cloud provider paths are wired:

- verify sidecar boot without local model cache and without cloud keys;
- verify Settings can load provider catalog and cache status without heavy imports;
- verify local missing-model state is actionable;
- verify cloud missing-consent/missing-key states are blocked before network;
- verify no raw audio, transcript text beyond immediate test result, API key, full local path, or raw provider exception appears in diagnostics/logs.

</tasks>
