# 19-01 Plan: STT Contracts, Registry, Cache, And Readiness Foundation

<objective>
Create the provider-neutral STT foundation: contracts, sidecar STT protocol, lightweight provider registry, app-managed model cache metadata, readiness gate state, and admin endpoint skeletons that do not load heavy providers at app boot.
</objective>

<execution_context>
Phase 19 depends on Phase 18, but Phase 17/18 execution may still be in progress. Before editing, inspect actual upstream outputs for audio config, provider catalog, diagnostics, and Settings bridge names. Extend existing files if they exist; do not create competing contracts or duplicate admin routes.

This plan intentionally does not implement real FunASR, faster-whisper, OpenAI, or Groq transcription. It creates the shared foundation those adapters plug into in 19-02 and 19-03.
</execution_context>

<tasks>

## 2. Extend Audio Contracts For STT

Extend `packages/contracts/py/contracts/audio_provider.py` with Pydantic types for:

- STT provider catalog entries, including provider id, display name, local/cloud kind, capability labels, recommended/default flags, required credential/consent state, supported language modes, and test support.
- local STT model catalog entries, including provider id, model id, display name, source label, approximate size label/bytes when known, cache path display, download status, remove availability, and loaded status.
- STT provider config, including selected provider, enabled flag, selected local model id/path, cache root, language mode, cloud provider endpoint/model id, provider-specific consent flags, and readiness stamp.
- readiness state with `health_check_passed`, `test_transcription_passed`, timestamps, invalidation fingerprint, `active_allowed`, and invalidation reason.
- test transcription request/result with base64 WAV input, duration, provider id, model/cache state, latency, normalized health/failure, and transcript output.
- download/remove operation request/result with progress-safe final status.

Contract rules:

- Do not add fields that require microphone chat UX, VAD, streaming preview, or final chat submission.
- Do not store raw test audio in any persistent contract.
- Cloud diagnostics contracts must not include transcript text except the immediate test result returned to Settings.
- Add or update contract tests to cover defaults, validation, readiness invalidation fields, and generated TS/schema outputs.

Run:

```powershell
npm run codegen:contracts
npm run check:contracts
```

## 3. Add STT Provider Protocol And Error Taxonomy

Add `sidecar/src/sidecar/stt/provider.py` with a narrow provider interface:

- `STTRequest`: normalized audio bytes or float PCM plus sample rate, duration, language mode, provider model id, and optional prompt/hints.
- `STTResult`: text, language, segments if available, latency, provider metadata, and redacted diagnostics.
- `STTProviderError`: provider id, health state/category, retryable flag, safe summary, redacted diagnostics.
- `STTProvider` protocol with `provider_id`, `health()`, `transcribe()`, `shutdown()`, and a lazy `ensure_loaded()` or equivalent.

Do not import provider-specific libraries in this base module.

## 4. Add Lightweight Registry And Catalog

Add `sidecar/src/sidecar/stt/registry.py`.

The registry should:

- list provider catalog metadata without importing heavy provider modules;
- mark FunASR/SenseVoiceSmall as the recommended local default;
- mark faster-whisper as local fallback;
- mark OpenAI/Groq as cloud and blocked until consent plus credentials;
- build providers lazily through import functions only when health/test/enable requires them;
- expose unsupported/missing-dependency health without crashing sidecar boot.

Add tests that monkeypatch provider imports and prove catalog/status calls do not import heavy modules.

## 5. Add App-Managed Model Cache Metadata

Add `sidecar/src/sidecar/stt/model_cache.py`.

It should:

- resolve an app-managed cache root from audio config or user data env;
- keep paths inside the intended app cache directory unless an explicit local model path override is supported;
- report cache path, existence, size, and status for FunASR/SenseVoiceSmall and faster-whisper model entries;
- provide safe remove behavior for app-managed model directories only;
- define download operation interfaces but leave real provider-specific downloads to 19-02.

Tests must verify path traversal protection and remove restrictions.

## 6. Add Readiness Gate Helper

Add `sidecar/src/sidecar/stt/readiness.py`.

It should:

- compute a stable fingerprint over provider id, selected model/cache path, language mode, cloud endpoint/model id, credential-present flag, and consent flag;
- mark readiness invalid if the fingerprint changes;
- require both health pass and a successful non-empty test transcript for `active_allowed`;
- mark a runtime provider failure as unhealthy/disabled until another passing test.

## 7. Extend Admin Audio Endpoints With STT Skeletons

Extend `sidecar/src/sidecar/admin/audio.py` with STT endpoints or route functions matching existing Phase 18 endpoint shape:

- provider catalog/status endpoint;
- local model status endpoint;
- model download/remove endpoints returning "not implemented/provider adapter pending" where 19-02 will fill behavior;
- health endpoint returning blocked/missing-model/missing-credential/unavailable states;
- test transcription endpoint accepting WAV payload but returning adapter-pending until 19-02/19-03 implement real providers;
- enable endpoint that refuses enablement until readiness permits it.

Every response must be redacted and must not retain raw audio.

## 8. Wire Lightweight STT Services At Sidecar Startup

In `sidecar/src/sidecar/ws/server.py`, add only lightweight STT registry/cache/readiness state to `app.state`.

Boot rule:

- no FunASR import;
- no faster-whisper import;
- no OpenAI/Groq SDK import;
- no torch/torchaudio import;
- no model download;
- no model load;
- no idle preload task.

Add a sidecar boot/admin test that proves `/health` and `/admin/audio/status` still work if provider libraries are absent.
