<task type="auto" tdd="true">
  <name>Task 1: Add main-process conversation persistence</name>
  <files>
    - apps/electron-main/src/conversation-store.ts
  </files>
  <behavior>
    - Store has `schemaVersion: 1`, `activeSessionId`, and session records.
    - If no session exists, helpers create one empty active session titled `New chat`.
    - Sessions persist locally and are not capped by count, age, or message count.
    - `commitConversationTurn` appends exactly one user message and one assistant message to the target session.
    - First completed turn auto-titles an unrenamed `New chat` session from the first user message.
    - Manual rename sets `titleSource: manual` and prevents future auto-title overwrites.
    - Delete-single chooses a valid next active session or creates a new empty active session when the last session is deleted.
    - Clear-all removes all prior sessions and creates a fresh empty active session.
  </behavior>
  <action>
    Implement a new typed main-process store module using `electron-store`, following the existing `window-store.ts` style. Keep transcript persistence separate from chrome/window preferences if practical, e.g. a named `conversation-history` store, so large message histories do not bloat the UI preference store.

    Add validation/normalization helpers:
    - trim manual titles and fall back to `New chat` for empty values
    - derive auto-title from the first user message without calling an LLM
    - clamp title length for display/storage consistency
    - generate stable ids with `crypto.randomUUID()` or a safe fallback
    - preserve message order and ISO timestamps

    Do not store embeddings, semantic memory, avatar-specific facts, retrieval metadata, cloud identifiers, or export/import metadata.
  </action>
  <verify>
    <automated>npm --workspace apps/renderer run typecheck</automated>
  </verify>
  <done>
    Main process has a durable local conversation store that satisfies complete-session CRUD and commit-turn semantics.
  </done>
</task>
