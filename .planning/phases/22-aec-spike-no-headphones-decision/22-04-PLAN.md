# 22-04 Plan: Structured UAT And Verdict Closure

<objective>
Close Phase 22 with structured real-world AEC/no-headphones evidence, set the shipped `Ready` / `Limited` / `Unsafe` status from the UAT verdict, and verify all AEC requirements.
</objective>

<execution_context>
This plan depends on 22-01 diagnostics, 22-02 self-speech guard, and 22-03 Settings policy. It must not retain raw audio as normal evidence and must not build a runtime automatic no-headphones classifier.
</execution_context>

<tasks>

## 1. Create UAT Document Skeleton

Create `.planning/phases/22-aec-spike-no-headphones-decision/22-UAT.md`.

Include:

- purpose and scope;
- explicit rule that raw microphone/TTS audio is not retained;
- environment section;
- test matrix;
- result fields;
- final verdict field: `Ready`, `Limited`, or `Unsafe`;
- limitations section;
- follow-up/gap section.

## 2. Define Environment Notes

The UAT environment section must capture:

- OS and app build mode;
- microphone device or practical description;
- speaker/headphone state;
- output volume;
- room noise;
- STT provider and provider readiness state;
- VAD enabled/disabled, sensitivity, and silence timeout;
- requested/applied AEC constraints from diagnostics;
- no-headphones status before test.

Do not require detailed driver versions unless execution discovers a real issue where they matter.

## 3. Define Test Matrix

Add manual checks for:

- microphone permission and AEC diagnostics are available;
- PTT no-headphones utterance while Teto is idle;
- VAD no-headphones utterance while Teto is idle;
- TTS-active no user speech: assistant output must not submit as user text;
- TTS-active user overlap with unrelated utterance: may submit if it passes guard;
- TTS-active assistant-overlap/self-speech candidate: must be discarded;
- local STT provider path;
- cloud STT path only if explicitly configured and consented, otherwise mark not tested;
- Settings status and override behavior;
- VAD conservative defaults and PTT availability.

Each row should record expected result, observed result, pass/fail, diagnostic reference, and notes.

## 4. Run Automated Regression

Run the targeted automated tests from 22-01 through 22-03:

- AEC diagnostics tests;
- self-speech guard tests;
- Settings status/default tests;
- Chat voice input tests;
- audio playback lifecycle tests;
- renderer typecheck;
- contract/schema drift check.

Fix failures inside Phase 22 scope only.

## 5. Execute Manual UAT

Run every UAT row using the real app, real microphone, and real TTS playback path.

Record:

- metadata and diagnostics only;
- no raw audio;
- no full discarded transcript text if it is assistant-like and redaction is required;
- whether each requirement `AEC-01` through `AEC-04` is satisfied.

If an environment dependency is missing, mark that row blocked with exact reason and keep the verdict conservative.

## 6. Set Shipped Status From Verdict

After UAT, update the configured default no-headphones status/copy to match the verdict:

- `Ready` only if strict repeatable evidence passed;
- `Limited` for minor/edge-case issues;
- `Unsafe` when UAT judgment says no-headphones behavior is unusable.

Do not create runtime logic that continuously changes the status automatically.

## 7. Write Verification Notes

Create `.planning/phases/22-aec-spike-no-headphones-decision/22-VERIFICATION-NOTES.md`.

Include:

- automated test commands and outcomes;
- UAT verdict;
- coverage table for `AEC-01` through `AEC-04`;
- known limitations;
- any blocked checks;
- confirmation that raw audio was not retained.

</tasks>
