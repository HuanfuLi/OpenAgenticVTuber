# 22-01 Plan: AEC Diagnostics And Status Model

<objective>
Add the shared no-headphones status model and renderer AEC diagnostics foundation so Phase 22 can request browser/WebRTC echo cancellation, record what the browser actually applied, and keep VAD defaults conservative.
</objective>

<execution_context>
Phase 22 depends on Phase 19-21 execution. Before editing, inspect actual Phase 19-21 summaries and files, especially the Phase 20 voice-capture controller and Settings voice input section. If the planned Phase 20 file names differ, use the actual names. Do not create a second microphone capture path.
</execution_context>

<tasks>

## 1. Discover Actual Voice Input Outputs

Read:

- `.planning/phases/19-stt-provider-abstraction-local-cloud-providers/*-SUMMARY.md`
- `.planning/phases/20-renderer-voice-capture-ptt-vad-preview-ux/*-SUMMARY.md`
- `.planning/phases/21-code-switch-evaluation-hardening/*-SUMMARY.md`
- `apps/renderer/src/audio/`
- `apps/renderer/src/state/`
- `apps/renderer/src/screens/Settings/`
- `packages/contracts/py/contracts/audio_provider.py`
- `apps/electron-main/src/safe-storage.ts`

Map actual names for:

- voice capture controller;
- VAD controller/settings;
- final STT submission path;
- Settings voice input component;
- audio config persistence.

Use those names throughout this plan.

## 2. Extend Shared Audio Contracts

In `packages/contracts/py/contracts/audio_provider.py`, add or extend voice input config with:

- `NoHeadphonesStatus = Literal["ready", "limited", "unsafe"]`;
- a configured no-headphones status default, initially conservative unless Phase 22 UAT later updates it;
- an explicit unsafe override boolean;
- AEC diagnostics fields that store metadata only, such as supported constraints, requested constraint, applied setting, sample rate/channel count if available, user agent/platform if already exposed safely, and last diagnostic timestamp;
- VAD tuning fields if Phase 20 did not already add them, including conservative initial sensitivity and silence timeout plus a marker that user tuning was learned from successful use.

Regenerate the TypeScript contract and JSON schema using the repo's existing contract generation command. If no generator script exists, update generated mirrors exactly as existing generated files expect and add a note to the plan summary during execution.

## 3. Update Persisted Defaults And Migration

Update `apps/electron-main/src/safe-storage.ts` so new audio fields have safe defaults:

- no-headphones status defaults to `unsafe` or the existing conservative equivalent until Phase 22 UAT changes it;
- unsafe override defaults to false;
- VAD remains disabled/opt-in if the field exists;
- sensitivity and silence timeout remain conservative;
- existing v2 configs migrate without losing prior TTS/STT settings.

Add or update safe-storage migration tests for old configs and default config.

## 4. Add AEC Diagnostic Helper

Create `apps/renderer/src/audio/aec-diagnostics.ts` or extend an equivalent actual Phase 20 module.

Implement helpers to:

- inspect `navigator.mediaDevices.getSupportedConstraints()`;
- choose capture audio constraints that request echo cancellation, preferring `"all"` only when safely supported and falling back to boolean `true`;
- inspect `MediaStreamTrack.getCapabilities()`, `getConstraints()`, and `getSettings()` where available;
- produce a redacted diagnostic object that records metadata only and never raw audio or transcript text.

The helper must tolerate browsers/Electron versions that do not expose every method or enum value.

## 5. Wire Diagnostics Into Existing Capture

Update the actual Phase 20 voice capture controller to:

- request AEC-capable constraints for no-headphones/VAD capture paths;
- keep PTT functional if constraints fall back;
- record diagnostic snapshots after a stream is obtained;
- clean up tracks exactly as Phase 20 already does.

Do not make no-headphones `Ready` because diagnostics look favorable. That decision belongs to UAT in 22-04.

## 6. Add Tests

Add renderer tests for:

- choosing echo cancellation constraints when supported;
- falling back when `"all"` is unsupported;
- producing diagnostics when methods are partially missing;
- not storing raw audio or transcript text in diagnostics;
- keeping VAD defaults conservative in default config.

Add Electron-main tests for safe-storage migration/defaults.

</tasks>
