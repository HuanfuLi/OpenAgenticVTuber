# 22-03 Plan: Settings Status, Override, And VAD Defaults

<objective>
Expose truthful no-headphones support status in Settings, preserve VAD as explicit opt-in with conservative defaults, and add the explicit Unsafe override path without changing baseline PTT availability.
</objective>

<execution_context>
This plan depends on the status/config model from 22-01 and the self-speech guard from 22-02. It should extend the actual Phase 20 Settings voice input component if present. If Phase 20 embedded voice settings directly in `Settings.tsx`, refactor only as much as needed to add Phase 22 status cleanly.
</execution_context>

<tasks>

## 1. Locate Or Create Voice Input Settings Surface

Inspect actual Settings implementation after Phase 20:

- `apps/renderer/src/screens/Settings/VoiceInputSection.tsx`
- `apps/renderer/src/screens/Settings/Settings.tsx`
- `apps/renderer/src/state/voice-input-store.ts`
- current copy constants in `apps/renderer/src/lib/copy.ts`

Use the existing Settings layout and avoid adding a Chat-facing no-headphones status surface.

## 2. Add Copy Constants

Add centralized copy for:

- no-headphones status header/label;
- `Ready`, `Limited`, `Unsafe`;
- concise recommendation for each status;
- Limited/Unsafe recommendation: PTT + headphones;
- Unsafe override label and warning;
- diagnostics/details link text.

Keep copy short and operational. Do not include full UAT notes inline.

## 3. Render Settings Status

In the voice input Settings section, show:

- current no-headphones status;
- concise recommendation;
- AEC diagnostics summary or a link/button to logs/diagnostics if diagnostics are already surfaced;
- unsafe override control only when status is `Unsafe`;
- VAD enablement disabled or guarded until override is explicit.

Do not add a dedicated AEC test wizard in this plan.

## 4. Enforce VAD Default Policy

Update the voice input state/config layer:

- VAD remains disabled by default for all statuses;
- `Ready` removes scary warning copy but does not auto-enable VAD;
- `Limited` keeps VAD off by default and shows warning that PTT + headphones are safer;
- `Unsafe` requires explicit override before VAD can be enabled in no-headphones mode;
- PTT remains available whenever STT readiness allows it;
- user-tuned sensitivity/silence timeout can be remembered after successful use, but initial defaults stay conservative.

If Phase 20 already has VAD controls, extend them rather than replacing them.

## 5. Persist Override And Learned Tuning

Wire Settings changes to existing config persistence:

- unsafe override persists through `saveStoredConfig`;
- learned VAD tuning persists only when user changes settings after successful use or through existing Settings controls;
- changing no-headphones status or override does not reset unrelated TTS/STT provider config.

## 6. Add Tests

Add renderer/settings tests for:

- Settings shows Ready/Limited/Unsafe labels;
- Limited and Unsafe recommend PTT + headphones;
- Chat does not render no-headphones status copy;
- Unsafe requires explicit override before VAD enablement;
- Ready does not auto-enable VAD;
- PTT remains available in voice controls when STT is ready even if no-headphones is Unsafe;
- status details remain concise and full UAT notes are not in Settings.

</tasks>
