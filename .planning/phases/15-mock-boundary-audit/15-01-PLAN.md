# 15-01 Plan: Audit and Enforce Mock Boundaries

<objective>
Finish v2.1 by removing dev mock dependencies from normal production user flows, replacing mock alert actions with real or truthful controls, and adding regression tests that keep mock/demo utilities isolated to development-only surfaces.
</objective>

<execution_context>
User chose to plan without a Phase 15 `CONTEXT.md`; this plan is based on `.planning/ROADMAP.md`, `.planning/REQUIREMENTS.md`, and the Phase 15 research scan.

Current confirmed leaks:

- `apps/renderer/src/state/app-store.tsx` imports `mockBanners` and `mockToasts` from `@/dev/__mocks__/mock-backend`.
- `apps/renderer/src/screens/Chat/Chat.tsx` imports `mockBanners` and `SCRIPTED_CONVO`, listens for `chat:inject`, and merges scripted messages into normal chat rendering.
- Normal UI still contains mock alerts:
  - VTube Studio docs in `Chat.tsx`.
  - setup help docs in `LLMSetup.tsx`.
  - open log folder in `LogsDrawer.tsx`.
  - open log folder in `Settings.tsx`.
- `DevPanel` itself may keep dev mock controls because it is mounted behind `import.meta.env.DEV`, but normal source must not depend on those mocks.
</execution_context>
