# 20-02 Plan: Renderer Voice Capture Controller And Settings Hotkey

<objective>
Build the renderer-side voice input foundation: microphone capture controller, WAV/chunk encoding, voice input state store, PTT hotkey Settings configuration, and conservative VAD settings state without adding Chat submission yet.
</objective>

<execution_context>
This plan depends on the bridge and contracts from 20-01. It should reuse the Phase 19/20 preload methods and should not call sidecar URLs directly.

This plan prepares Chat integration but does not render the Chat mic control or submit final transcripts. Those are 20-03.
</execution_context>

<tasks>

## 1. Add Renderer Voice Input State Store

Create `apps/renderer/src/state/voice-input-store.ts`.

State should include:

- readiness from `window.api.getVoiceInputReadiness`;
- capture state: idle, permission needed, listening, recording, previewing, finalizing, queued, error;
- latest transient preview transcript and sequence id;
- final transcript candidate not yet submitted;
- one pending queued final transcript slot for turn-in-progress handling;
- VAD settings: enabled, sensitivity, silence timeout, conservative defaults;
- PTT hotkey setting loaded from stored audio/settings state.

Do not persist preview transcript text.

## 2. Add Voice Capture Controller

Create `apps/renderer/src/audio/voice-capture.ts`.

Use:

- `navigator.mediaDevices.getUserMedia({ audio })` for permission/capture;
- `MediaRecorder` with a small preview timeslice for preview chunks;
- explicit track cleanup on stop, cancel, error, unmount, and sidecar reconnect;
- sequence ids so stale preview results are ignored;
- final transcription request after stop.

Audio constraints:

- mono where possible;
- request browser noise suppression/echo cancellation only as best-effort constraints;
- do not fail the whole capture if optional constraints are unsupported.

## 3. Add WAV Encoding Helper If Needed

If Phase 19 Settings recorder already provides a safe in-memory WAV encoder, reuse or move it. Otherwise add `apps/renderer/src/audio/wav-encoder.ts`.

Rules:

- encode only in memory;
- avoid storing audio blobs;
- return duration and base64 payload expected by 20-01 bridge;
- handle empty/too-short input as typed errors.

## 4. Add PTT Hotkey Settings

Extend the existing Voice Input Settings section or audio settings state with:

- configurable PTT shortcut;
- default shortcut chosen by implementation, documented in copy/tests;
- validation that avoids clashing with existing app-reserved shortcuts if such a list exists;
- Settings-only configuration UI.

Do not add inline Chat shortcut editing.

## 5. Add Conservative VAD Settings State

Add VAD configuration fields:

- enabled false by default;
- sensitivity default medium/conservative;
- silence timeout default conservative;
- no-headphones/AEC copy that stays experimental/deferred until Phase 22.

Actual VAD auto-submit behavior is implemented in 20-04; this plan only provides durable Settings state and store plumbing.

## 6. Add Unit Tests

Cover:

- `getUserMedia` is called only after explicit start/hold action;
- permission denied/no device errors become visible state;
- tracks stop on stop/cancel/error;
- preview sequence ids ignore stale results;
- preview transcript is held only in transient store;
- PTT hotkey setting saves through Settings and is not rendered inline in Chat;
- VAD defaults are disabled/conservative.

</tasks>
