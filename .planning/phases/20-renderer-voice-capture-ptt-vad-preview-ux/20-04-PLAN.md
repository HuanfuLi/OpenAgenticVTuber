# 20-04 Plan: VAD Auto-Submit, Conservative Safety, And Final UAT

<objective>
Complete Phase 20 by adding explicit VAD auto-submit, conservative VAD safety copy/defaults, full voice-input regression tests, and manual UAT evidence for all Phase 20 success criteria.
</objective>

<execution_context>
This plan depends on the Chat PTT and final submission path from 20-03. VAD should reuse the same final transcript submission and queueing behavior. It must not implement wake word, AEC, no-headphones claims, or barge-in cancellation.
</execution_context>

<tasks>

## 1. Add Conservative VAD Controller

Create `apps/renderer/src/audio/vad-controller.ts` or extend the 20-02 capture controller with a separate VAD module.

Behavior:

- VAD is disabled by default;
- user must explicitly enable VAD in Settings or Chat control if Settings already exposes the toggle;
- sensitivity and silence timeout are visible controls;
- default sensitivity is conservative;
- silence timeout avoids short accidental submits;
- during active TTS/speaking, VAD should not auto-submit assistant speech as user input.

Use browser Web Audio primitives for a lightweight volume/silence gate. If implementation needs real-time frame processing beyond simple analyser logic, use AudioWorklet rather than main-thread heavy loops.

## 2. Wire VAD To Existing Capture And Submit Paths

When VAD is enabled:

- start listening only after STT readiness and mic permission pass;
- begin recording when speech crosses threshold;
- finalize after configured silence timeout;
- send preview/final transcription through the same bridge used by PTT;
- submit final text through the same unchanged final text path used by PTT;
- apply the same turn-in-progress queue rules.

Do not add wake-word behavior.

## 3. Add User-Facing VAD State And Safety Copy

Update Chat and Settings copy:

- VAD enabled/disabled state;
- listening/recording/finalizing/error state;
- sensitivity and silence timeout labels;
- conservative no-headphones warning/status that does not claim AEC success;
- use-headphones/PTT fallback language if the app is speaking.

Keep copy concise and operational.

## 4. Add VAD And Integration Tests

Cover:

- VAD disabled by default;
- enabling VAD requires STT readiness;
- threshold crossing starts recording;
- silence timeout finalizes and auto-submits;
- sensitivity and timeout settings affect controller thresholds/timers;
- active speaking state blocks or queues auto-submit without playback interruption;
- no wake word state exists;
- final VAD transcript is submitted unchanged and preview is not persisted.

## 5. Final Phase Regression

Run the full relevant test set:

- renderer voice capture tests;
- VAD controller tests;
- Chat voice input tests;
- Chat streaming tests;
- Settings tests;
- renderer typecheck;
- Electron build;
- sidecar voice input endpoint tests.

Fix failures inside Phase 20 scope only.

## 6. Write Phase 20 UAT

Create `20-UAT.md` with manual checks for:

- microphone permission grant/deny;
- PTT hold-to-record preview and final submission;
- VAD enable, sensitivity, silence timeout, and auto-submit;
- preview text absent from conversation history;
- final text unchanged for Chinese/English/mixed input;
- active-turn queueing while Teto is replying/speaking;
- no no-headphones/AEC claim before Phase 22.

Document environment, provider used, and any blocked manual checks.

</tasks>
