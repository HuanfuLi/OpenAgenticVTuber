# 20-01 Plan: Voice Input Contracts, Permission, And Transcription Bridge

<objective>
Create the Phase 20 runtime bridge for Chat voice input: contracts for preview/final transcription, Electron microphone permission handling, preload IPC methods, and a sidecar admin route that uses the Phase 19 enabled STT provider without bypassing readiness.
</objective>

<execution_context>
Phase 20 depends on Phase 19. Before editing, inspect the actual Phase 19 outputs for STT catalog, readiness, test transcription, provider registry, and Electron bridge names. Extend existing files and endpoint groups; do not create duplicate STT provider contracts or a second provider registry.

This plan does not build the Chat UI, renderer capture controller, PTT hotkey handling, VAD, or final chat submission. It creates the typed bridge those plans consume.
</execution_context>

<tasks>

## 1. Discover Actual Phase 19 STT Surface

Read current files after Phase 19 execution:

- `packages/contracts/py/contracts/audio_provider.py`
- generated TS mirrors and JSON schemas
- `sidecar/src/sidecar/stt/provider.py`
- `sidecar/src/sidecar/stt/registry.py`
- `sidecar/src/sidecar/stt/readiness.py`
- `sidecar/src/sidecar/admin/audio.py`
- `apps/electron-main/src/ipc.ts`
- `apps/electron-main/preload/index.ts`
- `apps/electron-main/preload/index.d.ts`
- `apps/renderer/src/state/audio-settings.ts`

Map the actual selected/enabled provider and readiness result names. Use those names in all following edits.

## 2. Add Voice Input Runtime Contracts

Extend the existing audio/STT contracts with runtime voice input types:

- capture status enum: `idle`, `permission_needed`, `listening`, `recording`, `previewing`, `finalizing`, `queued`, `error`;
- runtime transcription request with base64 WAV or accepted Phase 19 audio payload, duration, sequence id, mode `preview` or `final`, and active session id if needed for diagnostics only;
- runtime transcription result with `ok`, `mode`, `sequence_id`, `transcript`, `is_final`, provider id, duration, latency, and redacted failure;
- readiness summary used by Chat voice controls, including whether STT is enabled, provider id, blocked reason, and setup destination.

Contract rules:

- Preview and final results are runtime-only; do not add persistence fields for preview text.
- Do not add translation, language normalization, wake word, or barge-in fields.
- Do not include raw audio or unredacted transcript/error diagnostics in stored config or logs.

Run:

```powershell
npm run codegen:contracts
npm run check:contracts
```

## 3. Add Electron Microphone Permission Handling

In Electron main startup/session setup, add microphone/media permission handling using Electron session handlers.

Rules:

- handle both permission request and permission check paths;
- scope allow behavior to the app renderer origin only;
- allow only media/microphone capture required by the app;
- deny unknown origins and unrelated permissions by default;
- expose enough state for renderer errors: denied, unavailable, no input device, and unexpected failure.

Add or extend Electron tests for allowed app-origin microphone requests and denied unrelated origins/permissions.

## 4. Add Preload And IPC Voice Input Methods

Extend `apps/electron-main/preload/index.ts`, `index.d.ts`, and `apps/electron-main/src/ipc.ts` with typed methods such as:

- `getVoiceInputReadiness()`;
- `requestMicrophonePermission()` or equivalent if renderer cannot rely on `getUserMedia` alone;
- `transcribeVoiceInput(input)` for preview and final runtime transcription.

IPC rules:

- renderer never calls the sidecar admin URL directly;
- IPC does not log base64 audio;
- sidecar-not-ready returns typed unavailable/readiness state;
- cloud STT still requires Phase 19 consent/credential/readiness; do not silently choose another provider.

## 5. Add Sidecar Runtime Transcription Endpoint

Extend `sidecar/src/sidecar/admin/audio.py` or the actual Phase 19 audio router with a runtime voice input endpoint.

Behavior:

- refuses when STT is disabled or readiness is not active;
- dispatches to the selected Phase 19 provider for preview/final transcription;
- normalizes provider failures into existing redacted health/failure contracts;
- never persists raw audio or transcripts;
- marks runtime provider failure unhealthy/disabled using the Phase 19 readiness failure path where available.

## 6. Add Contract, Electron, And Sidecar Tests

Cover:

- voice input contracts serialize preview and final results;
- permission handlers allow only the expected renderer origin/media permission;
- IPC returns sidecar unavailable safely;
- runtime transcription refuses when STT is not ready;
- runtime transcription uses selected provider only and does not auto-fallback;
- preview/final payloads do not include storage fields.
