# 20-07 Plan: Phase 19/20 STT Runtime Integration Fixes

<objective>
Close the integration gaps between Phase 19 STT provider/cache behavior and Phase 20 live voice capture: make Settings copy match real download support, make Chat readiness re-check local model presence, make model operations honor custom cache roots, and make preview transcription robust with real MediaRecorder output.
</objective>

<execution_context>
The Phase 19 gap closure added real explicit local model downloads, valid WAV Settings tests, coherent readiness, and off-event-loop provider work. The Phase 20 live path consumes that state through Electron IPC, renderer voice capture, and `/admin/audio/voice-input`.

The integration audit found four remaining gaps:

1. Settings still displays stale copy (`Automatic download unavailable`, `Provider adapter lands in Phase 19`) even though Phase 19 now has real download behavior.
2. A persisted readiness fingerprint can remain active after local model files are removed or deleted, so Chat can show PTT ready until runtime transcription fails.
3. Model catalog/status can use `config.cache_root`, but download/remove currently do not receive that root and can mutate the default cache instead.
4. Preview transcription encodes each MediaRecorder timeslice blob independently; real WebM/Opus chunks after the first may not be independently decodable.

This is a Phase 20 integration plan because the user-facing failure is live voice capture readiness/preview behavior, but it intentionally touches Phase 19 STT cache/readiness files where the boundary contract is wrong.
</execution_context>

<tasks>

## 1. Update Truthful Settings Copy For Real Download

Update `apps/renderer/src/lib/copy.ts` and affected tests.

Required behavior:

- replace `VOICE_IN_MODEL_DOWNLOAD: "Automatic download unavailable"` with copy that describes the now-real explicit action, such as `Download model`;
- replace `VOICE_IN_TEST_NOT_READY: "STT test did not run. Provider adapter lands in Phase 19."` with current-state copy, such as `Run an STT test before enabling Chat voice input.`;
- update `apps/renderer/tests/Settings.test.tsx` so it no longer asserts old unavailable/manual-setup text;
- keep failure summaries from the sidecar visible when real download fails due dependency/network/provider errors.

Do not overpromise that download always succeeds; the action is explicit and can fail with an actionable sidecar summary.

## 2. Make Voice Readiness Re-Check Local Model Presence

Update sidecar readiness logic so Phase 20 Chat cannot stay ready if the Phase 19 local model no longer exists.

Implementation rules:

- keep `validate_stt_readiness(config)` as the pure fingerprint check;
- add a sidecar helper used by `/admin/audio/voice-input/readiness` that combines `validate_stt_readiness(config)` with `_local_model_status(request, config)`;
- for local providers, if readiness is active but model status is not `downloaded`, return `VoiceInputReadiness` with:
  - `ready: false`;
  - `blocked_reason: "readiness_not_active"`;
  - `setup_destination: "voice_settings"`;
  - nested STT readiness invalidated with `invalidation_reason: "missing_model"`;
  - summary matching the existing missing-model runtime failure copy;
- ensure `/admin/audio/voice-input` and `/admin/audio/voice-input/readiness` share the same missing-model rule so the button state and runtime behavior agree.

Renderer follow-through:

- after removing a selected local model in Settings, clear or invalidate local `sttConfig.readiness` for that provider before notifying Chat readiness;
- refresh Chat readiness after download/remove just as save/test already does;
- do not mark readiness active after download alone. A passing STT test is still required.

Tests:

- add a sidecar test where config readiness is active but local cache is missing, and `/voice-input/readiness` returns blocked `missing_model`;
- add a Settings/Chat regression proving model remove refreshes or invalidates Chat readiness.

## 3. Make Download/Remove Cache-Root Aware Without Sending Full STT Config

Update the cache operation contract.

Preferred contract change:

- add optional `cache_root: string | null = None` to `STTModelCacheOperationRequest`;
- regenerate TypeScript and JSON schema through existing contract codegen;
- update Electron/renderer types and tests.

Implementation rules:

- `sidecar/src/sidecar/admin/audio.py` must construct `STTModelCache(cache_root=payload.cache_root)` for `/stt/models/download` and `/stt/models/remove`;
- `apps/renderer/src/screens/Settings/Settings.tsx` must pass `cache_root: sttConfig.cache_root ?? null` for download/remove;
- do not pass full `STTProviderConfig` to model operation endpoints, because it can contain cloud API keys;
- keep path-safety checks under the resolved cache root.

Tests:

- sidecar test: download/remove with a temporary custom cache root mutates that root, not the default root;
- renderer/electron tests: operation requests include only provider id, model id, and cache root, never cloud credentials.

## 4. Make Preview Transcription Robust With Real MediaRecorder Chunks

Update `apps/renderer/src/audio/voice-capture.ts`.

Problem:

- `ondataavailable` currently sends `event.data` directly to `transcribePreview(...)`;
- real MediaRecorder timeslice chunks, especially after the first WebM/Opus chunk, may not decode independently.

Required behavior:

- keep accumulating chunks in `this.chunks`;
- for preview, encode a blob made from accumulated chunks for the current capture id, not the isolated chunk;
- keep stale sequence gating intact so older preview results cannot overwrite newer previews;
- treat too-short/decode-failed preview attempts as non-fatal preview misses when recording is still active, not as a capture-level error that prevents finalization;
- final transcription continues using the combined final chunks.

Tests:

- update `apps/renderer/tests/voice-capture.test.ts` so the preview encoder receives accumulated blob content after multiple chunks;
- add a regression where one preview encode fails but final transcription still succeeds and submits/queues final text.

## 5. Update Integration UAT Evidence

Update `20-HUMAN-UAT.md` and, if needed, `20-UAT.md`.

Add explicit checks:
